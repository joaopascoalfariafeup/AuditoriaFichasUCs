Programação

Apreciação da Ficha da Unidade Curricular

Aspeto a Avaliar Apreciação Qualitativa Apreciação Quantitativa (0–3) (a)
Objetivos/resultados de aprendizagem Objetivos muito genéricos e vagos, sem verbos de ação específicos nem mapeamento claro para níveis cognitivos 1
Tópicos programáticos Programa bem estruturado e detalhado, cobrindo adequadamente programação imperativa e orientada a objetos 3
Metodologias de ensino e aprendizagem Metodologias adequadas com componente teórica e prática, mas descrição pouco detalhada do mapeamento 2
Metodologias de avaliação Sistema de avaliação bem estruturado com múltiplas componentes, mas sem mapeamento explícito para objetivos 2
Apreciação global Ficha com programa sólido mas objetivos insuficientemente detalhados 2.00

(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Programa muito bem estruturado e detalhado, cobrindo adequadamente tanto programação imperativa como orientada a objetos em C/C++
  • Inclusão de aspetos transversais importantes como boas práticas, documentação e validação de código
  • Sistema de avaliação diversificado com projeto de grupo (25%) e provas práticas em computador (75%)
  • Uso de ferramentas de correção automática para feedback rápido aos estudantes
  • Flexibilidade na avaliação com possibilidade de recurso que substitui as duas provas parciais
  • Bibliografia adequada e atualizada para a área de programação em C/C++
  • Carga horária realista e bem distribuída (56h contacto, 106h estudo autónomo)

Pontos fracos

  • Objetivos de aprendizagem extremamente vagos e genéricos, sem verbos de ação específicos
  • Ausência de mapeamento explícito entre objetivos, programa e avaliação
  • Falta de detalhamento sobre os níveis cognitivos visados (aplicação, análise, síntese)
  • Metodologias de ensino descritas de forma superficial, sem explicitar como contribuem para os objetivos
  • Ausência de critérios de avaliação específicos para o projeto de grupo
  • Inconsistências menores entre versões portuguesa e inglesa

Sugestões de melhoria

  • Reformular completamente os objetivos de aprendizagem usando verbos de ação específicos, por exemplo: "Implementar algoritmos usando estruturas de controlo em C/C++", "Desenvolver classes em C++ aplicando princípios de encapsulamento", "Aplicar conceitos de herança e polimorfismo na resolução de problemas"
  • Criar mapeamento explícito entre cada tópico programático e os objetivos de aprendizagem reformulados
  • Detalhar as metodologias de ensino explicando como cada tipo de aula contribui para objetivos específicos
  • Especificar critérios de avaliação para o projeto de grupo (funcionalidade, qualidade do código, documentação, etc.)
  • Incluir mapeamento das componentes de avaliação para os objetivos de aprendizagem
  • Corrigir inconsistências entre versões (por exemplo, "exam season" vs "época de recurso")
  • Considerar incluir uma componente de avaliação individual adicional para complementar o projeto de grupo
