Pontos fortes

  • Programa bem estruturado e abrangente, cobrindo adequadamente os temas fundamentais de sistemas de informação
  • Metodologias de ensino diversificadas, combinando teoria, prática e trabalho de projeto supervisionado
  • Sistema de avaliação detalhado e transparente, com componentes equilibradas entre avaliação supervisionada e trabalhos práticos
  • Inclusão de avaliação cruzada no trabalho de grupo, promovendo responsabilização individual
  • Bibliografia adequada e atual para a área
  • Carga de trabalho realista e coerente com os 6 ECTS (162 horas)
  • Coerência entre versões portuguesa e inglesa

Pontos fracos

  • Problemas graves de formatação com código HTML/XML desnecessário que prejudica significativamente a legibilidade
  • Objetivos gerais pouco específicos, sem uso de verbos de ação precisos da taxonomia de Bloom
  • Resultados de aprendizagem descritos de forma genérica, sem numeração ou estruturação clara
  • Ausência de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Falta de clareza sobre quais são as componentes T1 e T2 mencionadas na obtenção de frequência

Sugestões de melhoria

  • Limpar completamente o código HTML/XML desnecessário em todos os campos da ficha para melhorar drasticamente a legibilidade
  • Reformular os objetivos usando verbos de ação específicos (ex: "analisar", "projetar", "implementar", "avaliar") em vez de expressões genéricas como "preparar os estudantes"
  • Reestruturar os resultados de aprendizagem com numeração clara (O1, O2, etc.) e verbos de ação precisos, seguindo o modelo da ficha de referência
  • Adicionar mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem (ex: "→O1, O2")
  • Clarificar na secção "Obtenção de frequência" quais são as componentes T1 e T2, ou corrigir para referenciar as componentes corretas (miniteste e trabalho de grupo)
  • Considerar especificar melhor as metodologias de ensino, detalhando como as diferentes atividades contribuem para os objetivos específicos

Proposta de Nova Redação

Objetivos

Esta unidade curricular tem como objetivo capacitar os estudantes para projetar sistemas de informação adequados às necessidades dos utilizadores e aos objetivos de gestão das organizações. Os estudantes devem desenvolver competências em modelação conceptual de informação, modelação da interação com utilizadores, aplicação de metodologias de engenharia informática e utilização de sistemas de gestão de bases de dados relacionais.

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de: O1. Analisar problemas de gestão de informação em contexto organizacional. O2. Projetar modelos conceptuais de informação utilizando classes de objetos e modelo relacional. O3. Desenvolver modelos de interação com utilizadores aplicando princípios de usabilidade. O4. Aplicar metodologias de projeto de engenharia informática. O5. Implementar e testar protótipos de sistemas de informação com bases de dados relacionais. O6. Construir interfaces de utilizador, incluindo funcionalidades de pesquisa e produção de relatórios. O7. Avaliar a adequação de protótipos desenvolvidos face aos requisitos identificados. O8. Negociar requisitos para o desenvolvimento de sistemas empresariais robustos.

Programa

São abordados os seguintes tópicos programáticos, mapeados para os objetivos de aprendizagem: 1. PROJETO DE SISTEMAS DE INFORMAÇÃO: Introdução aos processos de análise e modelação de sistemas e organizações; Introdução ao projeto de construção de um SI (→O1, O4). 2. MODELAÇÃO DA INTERFACE COM O UTILIZADOR: Introdução ao processo; Introdução às formas de especificação e às ferramentas de prototipificação; Recomendações, princípios e regras para conceção de interfaces; Principais modelos de interação: menus, formulários, caixas de diálogo e manipulação direta; Problemas sobre modelação da interação com utilizadores (→O3, O6). 3. MODELAÇÃO CONCEPTUAL DE CLASSES: O Processo de Modelação Estática de Classes (UML); Classes de Objetos, Atributos e Métodos; Ligações, Associações e Agregações Simples; Generalização e Herança Simples; Mapeamento entre o Modelo de Classes e o Modelo Relacional (→O2, O5). 4. PROJETO DE BASES DE DADOS: Introdução aos SGBD - Sistemas de Gestão Bases de Dados; Linguagem SQL; Álgebra e Cálculo Relacional, Normalização Funcional de Dados para Conceção de BD (→O2, O5, O6). 5. CONCLUSÕES E REFERÊNCIAS: Avaliação de protótipos e negociação de requisitos para sistemas empresariais (→O7, O8).

Obtenção de frequência

Para acesso ao exame final, os estudantes têm de: - cumprir a regra de assiduidade (não faltar a mais de 25% das aulas, exceto se estiverem dispensados de frequência ao abrigo de estatutos especiais); - realizar as componentes de avaliação distribuída (miniteste de Access e trabalho de grupo).
