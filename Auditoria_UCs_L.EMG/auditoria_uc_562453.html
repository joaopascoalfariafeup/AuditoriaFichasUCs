Pontos fortes

  • Abordagem e-learning adequada ao contexto de uma UC de 1,5 ECTS
  • Foco em competências práticas relevantes (Python para análise de dados)
  • Bibliografia adequada e acessível
  • Carga horária coerente com os créditos ECTS (40,5h para 1,5 ECTS)
  • Coerência entre versões portuguesa e inglesa

Pontos fracos

  • Objetivos e resultados de aprendizagem demasiado genéricos e vagos
  • Programa superficial sem detalhamento dos conteúdos específicos
  • Sistema de avaliação contraditório e confuso
  • Ausência de alinhamento explícito entre objetivos, programa e avaliação
  • Critério de frequência inadequado (85% é uma classificação, não condição de acesso)

Sugestões de melhoria

  • Reformular os resultados de aprendizagem usando verbos de ação específicos (ex: "aplicar", "implementar", "analisar") e especificar competências concretas
  • Detalhar o programa com tópicos específicos de cada módulo (ex: estruturas de dados, manipulação de DataFrames, visualização, etc.)
  • Corrigir completamente o sistema de avaliação, definindo pesos claros para avaliação distribuída e exame final
  • Reformular as condições de frequência, substituindo "classificação superior a 85%" por critérios de participação adequados
  • Estabelecer mapeamento explícito entre objetivos, tópicos programáticos e componentes de avaliação
  • Clarificar se existem notas mínimas nas componentes de avaliação

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:
O1. Aplicar conceitos fundamentais de programação em Python para manipulação de dados.
O2. Utilizar bibliotecas Python (pandas, numpy, matplotlib) para importar, processar e visualizar dados.
O3. Implementar soluções para problemas de análise de dados usando séries temporais da Pordata e INE.
O4. Integrar Python com ferramentas de produtividade (MS-Excel) para análise de dados.
O5. Interpretar e comunicar resultados de análises de dados através de visualizações apropriadas.

Programa

O programa está estruturado em cinco módulos sequenciais:
Módulo 1: Fundamentos de Python - sintaxe básica, estruturas de dados, controlo de fluxo (→O1)
Módulo 2: Bibliotecas NumPy e Pandas - arrays, DataFrames, operações de manipulação de dados (→O2)
Módulo 3: Visualização de dados - biblioteca Matplotlib, criação de gráficos e dashboards (→O2, O5)
Módulo 4: Integração com Excel - importação/exportação, automatização de tarefas (→O4)
Módulo 5: Projeto prático - análise de séries temporais da Pordata/INE (→O3, O5)

Fórmula de cálculo da classificação final

A avaliação compreende as seguintes componentes:

1) Avaliação distribuída (AD) (Peso 40%):
- Quizzes online: 5 testes de transição entre módulos (Peso: 40%; →O1-O5)

2) Exame final (EF) (Peso 60%):
- Exame prático em laboratório de informática, duração 1h30m (Peso: 60%; Nota mínima: 8,0 valores; →O1-O5)

A classificação final (CF) é calculada como:
CF = round(0.40 × AD + 0.60 × EF; 0)

Obtenção de frequência

Para acesso ao exame final, os estudantes têm de:
- Completar todos os 5 módulos da plataforma e-learning;
- Realizar todos os quizzes de transição entre módulos.
