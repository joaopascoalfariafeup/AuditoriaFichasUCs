Pontos fortes

  • Metodologias de ensino bem adaptadas ao formato e-learning com diversidade de atividades (vídeos, exercícios, quizzes, sessões de apoio)
  • Foco em competências práticas relevantes (Python, bibliotecas de análise de dados, interface com Excel)
  • Bibliografia adequada e acessível através da biblioteca da universidade
  • Carga horária realista para 1,5 créditos ECTS (40,5 horas)
  • Abordagem progressiva por módulos sequenciais

Pontos fracos

  • Contradição crítica entre a descrição de "avaliação distribuída" e a fórmula final que considera apenas "100% exame"
  • Critério de 85% para progressão entre módulos é excessivamente elevado e pode ser desmotivador
  • Resultados de aprendizagem formulados como competências genéricas, sem verbos de ação específicos
  • Programa muito genérico, sem detalhamento dos conteúdos específicos de cada módulo
  • Ausência de mapeamento explícito entre objetivos, programa e avaliação
  • Falta de clareza sobre o peso real da avaliação distribuída (quizzes)

Sugestões de melhoria

  • Corrigir a contradição na avaliação: se há quizzes obrigatórios para progressão, estes devem ter peso na classificação final (ex: 30% quizzes + 70% exame)
  • Reduzir o critério de progressão de 85% para 60-70%, mais adequado para uma UC introdutória
  • Reformular os resultados de aprendizagem com verbos de ação específicos (ex: "Aplicar bibliotecas Python...", "Analisar dados...", "Criar visualizações...")
  • Detalhar o programa especificando os conteúdos de cada um dos cinco módulos mencionados
  • Clarificar as condições de obtenção de frequência (atualmente apenas menciona "classificação superior a 85% no teste de transição")
  • Especificar se a avaliação de recurso mantém o mesmo formato e critérios
  • Corrigir inconsistências entre versões PT/EN (ex: "Distributed evaluation" vs "Avaliação distribuída")

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:

  • Aplicar a linguagem Python e suas principais bibliotecas (pandas, numpy, matplotlib) na análise de dados
  • Analisar e processar conjuntos de dados de grande volume, nomeadamente séries temporais da Pordata e INE
  • Criar visualizações de dados utilizando bibliotecas Python apropriadas
  • Integrar Python com ferramentas de folha de cálculo (MS-Excel) para gestão de dados
  • Resolver problemas práticos de análise de dados aplicando os conceitos e técnicas aprendidos

Programa

O programa está estruturado em cinco módulos sequenciais:

  • Módulo 1: Fundamentos de programação em Python (sintaxe, estruturas de dados, controlo de fluxo)
  • Módulo 2: Bibliotecas essenciais para análise de dados (numpy para computação numérica, pandas para manipulação de dados)
  • Módulo 3: Visualização de dados com matplotlib e outras bibliotecas gráficas
  • Módulo 4: Interface Python-Excel (leitura, escrita e manipulação de ficheiros Excel)
  • Módulo 5: Análise prática de dados reais (séries temporais da Pordata e INE, casos de estudo)

Fórmula de cálculo da classificação final

A avaliação compreende duas componentes:

  • Avaliação distribuída (30%): Quizzes online realizados ao longo dos módulos para verificar a progressão na aprendizagem
  • Exame final (70%): Prova prática em sala de computadores com duração de 1h30m, focada na resolução de problemas de análise de dados

Fórmula: Classificação Final = 0,30 × Média dos Quizzes + 0,70 × Exame Final

A classificação final será expressa numa escala numérica inteira de 0 a 20 valores.

Obtenção de frequência

Para acesso ao exame final, os estudantes devem:

  • Completar todos os módulos da UC, obtendo classificação mínima de 60% em cada quiz de transição
  • Participar nas atividades de avaliação distribuída
