Pontos fortes

  • Programa bem estruturado e abrangente, cobrindo adequadamente os tópicos essenciais de data mining
  • Bibliografia adequada e atual, incluindo referência do próprio docente
  • Estrutura de avaliação equilibrada (50% exame + 50% trabalho) apropriada para UC mista
  • Notas mínimas (7.0) adequadas para garantir competências mínimas em ambas as componentes
  • Carga horária total correta (162h para 6 ECTS)

Pontos fracos

  • Resultados de aprendizagem formulados de forma muito genérica, sem verbos de ação específicos da taxonomia de Bloom
  • Metodologias de ensino descritas de forma muito sucinta e pouco informativa
  • Inconsistência entre versões PT/EN nas condições de obtenção de frequência
  • Falta de clareza sobre como recuperar a componente de trabalho em caso de insucesso
  • Ausência de mapeamento explícito entre tópicos programáticos e resultados de aprendizagem

Sugestões de melhoria

  • Reformular os resultados de aprendizagem usando verbos de ação específicos (ex: "aplicar algoritmos de clustering", "avaliar modelos de classificação", "implementar pipelines de pré-processamento")
  • Expandir a descrição das metodologias de ensino, especificando como se articulam aulas teóricas, exercícios práticos e desenvolvimento do projeto
  • Harmonizar as condições de obtenção de frequência entre as versões PT e EN, clarificando se existe obrigatoriedade de assiduidade
  • Especificar como os estudantes podem recuperar a classificação do trabalho (ex: novo trabalho no ano seguinte)
  • Adicionar mapeamento entre tópicos programáticos e resultados de aprendizagem para maior clareza pedagógica
  • Clarificar se existe exame de recurso para a componente de trabalho ou apenas para o exame final

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:
O1. Identificar e caracterizar problemas que podem ser resolvidos através de técnicas de data mining.
O2. Aplicar metodologias estruturadas (ex: CRISP-DM) para resolver problemas de data mining.
O3. Implementar e configurar algoritmos de clustering, classificação e regressão usando Python.
O4. Avaliar criticamente o desempenho de modelos preditivos usando métricas apropriadas.
O5. Aplicar técnicas de pré-processamento e preparação de dados.
O6. Desenvolver sistemas de recomendação e aplicar técnicas de frequent pattern mining.
O7. Interpretar e comunicar resultados de análise de dados tanto numa perspetiva técnica como de domínio de aplicação.

Métodos de ensino e atividades de aprendizagem

A UC combina aulas teórico-práticas com desenvolvimento de projeto:
- Aulas teóricas para apresentação e discussão dos conceitos fundamentais de data mining
- Sessões práticas para implementação de algoritmos e técnicas usando Python
- Desenvolvimento de projeto em grupo ao longo do semestre, com acompanhamento tutorial
- Apresentações orais dos projetos para discussão e partilha de experiências
O projeto permite aplicar os conhecimentos adquiridos a um problema real, seguindo uma metodologia estruturada desde a análise exploratória até à avaliação de resultados.

Obtenção de frequência

Para acesso ao exame final, os estudantes devem:
- Cumprir a regra de assiduidade (não faltar a mais de 25% das aulas práticas)
- Integrar um grupo de projeto e participar efetivamente no seu desenvolvimento
