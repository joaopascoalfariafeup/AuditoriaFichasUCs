Pontos fortes

  • Metodologia de ensino bem estruturada, combinando aulas tutoriais e laboratoriais de forma adequada à natureza prática da UC
  • Abordagem baseada em projeto (80% da avaliação) coerente com os objetivos de aprendizagem prática
  • Inclusão de avaliação individual (mini-testes) para verificar conhecimentos teóricos
  • Bibliografia adequada e atual, incluindo referências clássicas de engenharia de software
  • Carga de trabalho realista e bem distribuída (162h = 6 ECTS × 27h)
  • Reconhecimento da natureza laboratorial da UC nas condições de frequência

Pontos fracos

  • Programa demasiado genérico e sucinto, não permitindo compreender o conteúdo específico abordado
  • Resultados de aprendizagem excessivamente detalhados (14 itens) e com verbos pouco específicos
  • Ausência de mapeamento entre tópicos programáticos e resultados de aprendizagem
  • Inconsistências entre versões PT e EN nas condições de aprovação
  • Redação confusa das condições mínimas de aprovação
  • Erro tipográfico na versão inglesa ("sstudent")

Sugestões de melhoria

  • Consolidar os 14 resultados de aprendizagem em 6-8 itens mais abrangentes, usando verbos mais específicos da taxonomia de Bloom
  • Expandir o programa com tópicos mais específicos e detalhados, incluindo mapeamento explícito para os resultados de aprendizagem
  • Clarificar as condições de aprovação: substituir "A aprovação na unidade curricular está condicionada à obtenção de 40% em: - média das avaliações dos mini-testes (M1 + M2) - avaliação to projeto (PROJ" por "Para aprovação, é necessário obter classificação mínima de 8.0 valores (40%) tanto na média dos mini-testes (M1+M2)/2 como no projeto (PROJ)"
  • Corrigir inconsistências entre versões: na versão inglesa, alterar "aggregate of (M1 + M2)" para "average of (M1 + M2)" para ser coerente com a versão portuguesa
  • Corrigir erro tipográfico na versão inglesa: "sstudent" → "student"
  • Adicionar informação sobre como será feita a avaliação individual dentro das equipas de projeto

Proposta de Nova Redação

Resultados de aprendizagem e competências

Ao completar esta unidade curricular o estudante deve ser capaz de:

  1. Analisar e documentar requisitos de sistemas de software utilizando user stories e requisitos suplementares.
  2. Criar protótipos de interface com o utilizador e protótipos verticais para validação de arquitetura.
  3. Aplicar UML para modelação conceptual do domínio, descrição de arquitetura, estrutura de classes e comportamento de objetos.
  4. Desenvolver sistemas de software baseados no paradigma de Orientação por Objetos, incluindo testes unitários.
  5. Utilizar ferramentas de desenvolvimento colaborativo (IDE, controlo de versões, ALM).
  6. Planear, estimar e gerir processos ágeis de desenvolvimento de software.

Programa

São abordados os seguintes tópicos programáticos, mapeados para os objetivos de aprendizagem:

  • T1. Engenharia de Requisitos: identificação e documentação de requisitos funcionais com user stories; especificação de requisitos suplementares (→O1).
  • T2. Prototipagem: desenvolvimento de protótipos de interface gráfica; criação de protótipos verticais para validação de arquitetura (→O2).
  • T3. Modelação UML: diagramas de classes para modelação conceptual e estrutural; diagramas de sequência e estados para modelação comportamental; diagramas de arquitetura (→O3).
  • T4. Programação Orientada por Objetos: conceitos avançados (coleções, genéricos, interfaces gráficas, I/O, concorrência); desenvolvimento de sistemas OO; testes unitários (→O4).
  • T5. Ferramentas de Desenvolvimento: utilização de IDEs; sistemas de controlo de versões; ferramentas de Application Lifecycle Management (→O5).
  • T6. Processos Ágeis: conceitos, metodologias e práticas; planeamento e estimação; gestão de projetos de software (→O6).

Fórmula de cálculo da classificação final

A avaliação compreende as seguintes componentes:

  • PROJ: Projeto de software desenvolvido em equipa (Peso: 80%)
  • M1: Primeiro mini-teste individual (Peso: 10%)
  • M2: Segundo mini-teste individual (Peso: 10%)

A classificação final é calculada pela fórmula: CF = 0.80 × PROJ + 0.10 × M1 + 0.10 × M2

Para aprovação na UC, é necessário obter classificação mínima de 8.0 valores (40%) tanto na média dos mini-testes [(M1+M2)/2] como no projeto (PROJ).

A classificação do projeto pode variar entre elementos da mesma equipa, com base na participação presencial, autoavaliação e heteroavaliação realizadas internamente por cada equipa.
