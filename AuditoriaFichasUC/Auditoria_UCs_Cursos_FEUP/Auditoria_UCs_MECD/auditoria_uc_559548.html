Pontos fortes

  • Resultados de aprendizagem bem formulados com verbos de ação específicos e competências técnicas claramente definidas
  • Programa detalhado e bem estruturado, cobrindo adequadamente os temas essenciais de otimização matemática
  • Metodologia de ensino equilibrada entre teoria e prática, com uso de software especializado (CPLEX, ILOG Solver)
  • Sistema de avaliação diversificado com múltiplas componentes (testes, trabalhos, apresentação)
  • Flexibilidade na avaliação (escolha dos 2 melhores testes de 3 realizados)
  • Nota mínima de 40% em todas as componentes garante qualidade da aprendizagem
  • Carga horária total correta (168h = 6 ECTS × 28h, próximo das 27h/ECTS recomendadas)
  • Bibliografia adequada e específica da área
  • Versões PT/EN coerentes

Pontos fracos

  • Objetivos gerais demasiado sucintos e genéricos, não especificando claramente o contexto de aplicação
  • Falta de mapeamento explícito entre tópicos programáticos e resultados de aprendizagem
  • Descrição das metodologias de ensino poderia ser mais detalhada sobre as atividades específicas
  • Inconsistência na descrição da avaliação (menciona "testes" e "quizzes" alternadamente)
  • Carga horária ligeiramente acima do recomendado (168h vs 162h)

Sugestões de melhoria

  • Expandir a secção "Objetivos" para incluir o contexto de aplicação dos sistemas analíticos de apoio à decisão e sua relevância na engenharia e ciência de dados
  • Adicionar mapeamento explícito entre tópicos programáticos e resultados de aprendizagem (ex: "T1. Formulações de programação linear (→competência para construir modelos)")
  • Detalhar melhor as metodologias de ensino, especificando tipos de exercícios, duração das apresentações e critérios de avaliação das mesmas
  • Uniformizar a terminologia de avaliação, usando consistentemente "testes" ou "quizzes" em ambas as versões
  • Ajustar ligeiramente a carga horária para 162h (ex: reduzir "Elaboração de projeto" de 60h para 54h)
  • Clarificar se os "testes com consulta" permitem consulta a todos os materiais ou apenas a alguns específicos

Proposta de Nova Redação

Objetivos

Esta unidade curricular tem como objetivo introduzir os estudantes aos fundamentos teóricos e práticos de duas das principais ferramentas para a resolução exata de problemas de otimização em contextos de apoio à decisão:

  • Programação matemática – ideal para problemas com decisões contínuas e combinatórias;
  • Programação por restrições – ideal para problemas combinatórios, especialmente quando o espaço de soluções é reduzido.

A UC visa capacitar os estudantes para aplicar estas ferramentas na resolução de problemas reais de engenharia e ciência de dados, desenvolvendo competências essenciais para a conceção de sistemas analíticos de apoio à decisão.

Programa

São abordados os seguintes tópicos programáticos, mapeados para os resultados de aprendizagem:

T1. Formulações de programação linear para problemas de otimização linear contínua (→identificar, analisar e construir modelos).

T2. Análise geométrica de problemas de otimização (→estruturar problemas de otimização).

T3. Formulações de programação inteira (→construir modelos para problemas combinatórios).

T4. Resolução de problemas de programação linear: o Algoritmo Simplex (→obter soluções usando método simplex).

T5. Análise de sensibilidade (→analisar robustez das soluções).

T6. Dualidade em programação linear (→aplicar teoria da dualidade).

T7. Programação não-linear (→resolver problemas não-lineares).

T8. Resolução de problemas de programação inteira (→usar algoritmos de pesquisa em árvore).

T9. Utilização de IBM ILOG CPLEX Optimization Studio (→usar CPLEX através de Python).

T10. Programação por restrições (→usar ILOG Solver para problemas combinatórios).
