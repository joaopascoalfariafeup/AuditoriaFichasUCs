Pontos fortes

  • Tema relevante e atual (análise de dados em Python) adequado ao contexto académico
  • Abordagem e-learning bem estruturada com diferentes tipos de atividades (vídeos, exercícios, quizzes, sessões de apoio)
  • Bibliografia adequada e acessível através da biblioteca da universidade
  • Carga horária realista para 1,5 ECTS (40,5 horas)
  • Referência específica a fontes de dados reais (Pordata e INE)

Pontos fracos

  • Resultados de aprendizagem vagos, sem verbos de ação específicos da taxonomia de Bloom
  • Programa excessivamente genérico, sem detalhamento dos conteúdos específicos
  • Sistema de avaliação contraditório (indica "avaliação distribuída com exame final" mas depois "100% exame")
  • Critério de frequência inadequado (85% é uma classificação, não um critério de frequência)
  • Ausência de mapeamento entre objetivos, programa e avaliação
  • Falta de clareza sobre as componentes de avaliação e seus pesos

Sugestões de melhoria

  • Reformular os resultados de aprendizagem usando verbos de ação específicos (ex: "aplicar", "analisar", "criar") e tornando-os mais mensuráveis
  • Detalhar o programa especificando os conteúdos de cada um dos cinco módulos mencionados
  • Clarificar o sistema de avaliação: definir se existe realmente avaliação distribuída ou se é apenas exame final
  • Corrigir o critério de frequência, substituindo "classificação superior a 85%" por critérios adequados como assiduidade ou participação nas atividades obrigatórias
  • Estabelecer mapeamento explícito entre objetivos de aprendizagem, tópicos programáticos e componentes de avaliação
  • Especificar claramente os pesos das diferentes componentes de avaliação se existir avaliação distribuída

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:

  • Aplicar conceitos básicos de programação em Python para manipulação de dados
  • Utilizar bibliotecas Python (pandas, numpy, matplotlib) para análise e visualização de dados
  • Importar e processar dados de fontes externas, incluindo ficheiros Excel e bases de dados públicas
  • Analisar séries de dados disponíveis na Pordata e no INE para resolver problemas práticos
  • Criar visualizações básicas para comunicar resultados de análises de dados

Programa

O programa está estruturado em cinco módulos sequenciais:

  • Módulo 1: Fundamentos de Python - sintaxe básica, estruturas de dados, controlo de fluxo
  • Módulo 2: Biblioteca pandas - manipulação de DataFrames, limpeza e transformação de dados
  • Módulo 3: Biblioteca numpy - operações numéricas e estatísticas básicas
  • Módulo 4: Visualização com matplotlib - criação de gráficos e dashboards simples
  • Módulo 5: Integração com Excel e análise de dados reais da Pordata e INE

Fórmula de cálculo da classificação final

A avaliação compreende as seguintes componentes:

  • Avaliação distribuída (AD): Quizzes online no final de cada módulo (Peso: 40%)
  • Exame final (EF): Prova prática em laboratório de informática com duração de 1h30m (Peso: 60%)

Classificação final (CF) = 0,40 × AD + 0,60 × EF, arredondada à unidade

Obtenção de frequência

Para acesso ao exame final, os estudantes devem:

  • Completar todos os cinco módulos da UC
  • Realizar todos os quizzes de avaliação distribuída
  • Participar em pelo menos uma sessão de apoio (presencial ou online)
