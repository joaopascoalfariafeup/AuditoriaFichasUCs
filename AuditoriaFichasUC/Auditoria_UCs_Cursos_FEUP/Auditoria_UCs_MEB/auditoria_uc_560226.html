(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Programa bem estruturado e detalhado, com progressão lógica dos conceitos desde modelação UML até implementação web
  • Objetivos gerais claros e adequados ao contexto de um mestrado em Bioengenharia
  • Bibliografia adequada e atual para a área de sistemas de informação e bases de dados
  • Carga de trabalho realista (162h = 27h × 6 ECTS) com distribuição equilibrada entre componentes
  • Versões PT/EN coerentes e bem traduzidas
  • Software de apoio adequado e atual (SQLite, PHP, Docker)

Pontos fracos

  • Resultados de aprendizagem mal estruturados, com verbos de ação pouco específicos e sem seguir a taxonomia de Bloom
  • Metodologias de ensino descritas de forma muito sucinta, sem detalhes sobre atividades específicas
  • Condições de frequência incluem classificações mínimas (8 valores no projeto e exame), o que é inadequado
  • Ausência de mapeamento explícito entre objetivos, programa e metodologias
  • Projeto com peso muito elevado (50%) sem componentes de avaliação supervisionada intercalares

Sugestões de melhoria

  • Reformular os "Resultados de aprendizagem e competências" usando verbos de ação específicos da taxonomia de Bloom (ex: "analisar", "desenhar", "implementar", "avaliar") e organizá-los de forma mais estruturada
  • Expandir a secção "Métodos de ensino e atividades de aprendizagem" detalhando as atividades específicas de cada tipo de aula e como se alinham com os objetivos
  • Remover as classificações mínimas das condições de frequência, mantendo apenas: "Completar e apresentar o projeto" e "Realizar o exame"
  • Considerar adicionar uma componente de avaliação distribuída intercalar (ex: mini-teste sobre modelação UML/relacional) para equilibrar a avaliação
  • Incluir mapeamento explícito entre os tópicos programáticos e os resultados de aprendizagem
  • Clarificar se existe nota mínima nas componentes de avaliação (projeto e exame) para aprovação final

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:

  • Analisar requisitos de sistemas de informação e identificar necessidades de dados e processos de negócio
  • Desenhar modelos de dados usando diagramas de classes UML com classes, associações e atributos
  • Transformar modelos UML em esquemas relacionais eficientes, aplicando regras de mapeamento
  • Implementar e explorar bases de dados relacionais usando SQLite
  • Construir interrogações SQL para definição, consulta e manipulação de dados
  • Desenvolver interfaces web usando HTML5 e CSS para apresentação de dados
  • Implementar lógica de negócio e conectividade à base de dados usando PHP
  • Avaliar e aplicar conceitos básicos de sistemas de recomendação em aplicações web
  • Configurar e executar aplicações web usando containers Docker
  • Integrar todos os componentes num sistema de informação web funcional

Métodos de ensino e atividades de aprendizagem

A unidade curricular combina aulas teóricas e práticas numa abordagem integrada:

  • Aulas teóricas: Exposição dos conceitos fundamentais (modelação UML, modelo relacional, SQL, tecnologias web), acompanhada de exemplos práticos e discussão de casos de estudo reais
  • Aulas práticas: Resolução de exercícios de modelação e implementação, apoio tutorial ao desenvolvimento do projeto, e sessões de esclarecimento de dúvidas técnicas
  • Projeto prático: Desenvolvimento incremental de um sistema de informação web completo, desde a análise de requisitos até à implementação, permitindo aplicar todos os conceitos aprendidos
  • Apresentações: Apresentação e discussão dos projetos desenvolvidos, promovendo a partilha de experiências e soluções entre estudantes

Obtenção de frequência

Para acesso ao exame final, os estudantes têm de:

  • Cumprir a regra de assiduidade (não faltar a mais de 25% das aulas, exceto se estiverem dispensados de frequência ao abrigo de estatutos especiais)
  • Completar e apresentar o projeto nas duas entregas previstas: relatório com modelos UML e relacional (30%) e implementação do website (70%)
