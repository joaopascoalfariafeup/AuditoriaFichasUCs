Pontos fortes

  • Programa muito detalhado e bem estruturado, cobrindo adequadamente os tópicos essenciais de desenho e teste de software orientado a objetos
  • Metodologia de ensino apropriada com progressão lógica: exercícios práticos (7 semanas) seguidos de projeto integrador (6 semanas)
  • Sistema de avaliação diversificado com componentes práticas (participação, projeto) e teóricas (provas individuais)
  • Bibliografia adequada e atual, incluindo referências clássicas da área
  • Carga de trabalho realista e bem distribuída (162h = 6 ECTS × 27h)
  • Versões PT/EN coerentes entre si

Pontos fracos

  • Inconsistência entre objetivos detalhados (9 pontos) e resultados de aprendizagem mais genéricos (5 pontos)
  • Ponto 9 dos objetivos ("Patterns of Enterprise Applications") está incompleto/truncado
  • Falta de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Condições de frequência pouco claras quanto à participação "ativa" no projeto
  • Não especifica se as provas individuais permitem consulta ou não

Sugestões de melhoria

  • Completar o ponto 9 dos objetivos: "Aplicar padrões de arquitetura empresarial no desenho de aplicações"
  • Alinhar os resultados de aprendizagem com os objetivos, tornando-os mais específicos e usando verbos de ação adequados (ex: "aplicar", "identificar", "desenhar")
  • Adicionar mapeamento entre tópicos programáticos e objetivos (ex: "T1. Git → O1; T3. Java → O2")
  • Clarificar as condições de frequência: especificar o que constitui "participação ativa" no projeto
  • Especificar se as provas individuais são com ou sem consulta
  • Clarificar se a nota mínima de 40% se aplica à classificação final de cada componente ou às classificações parcelares

Proposta de Nova Redação

Objetivos

Pretende-se, nesta unidade curricular, que os alunos desenvolvam competências no desenho de aplicações usando o paradigma orientado a objetos. Pretende-se que os estudantes que obtenham aprovação à unidade curricular, consigam:

  1. Usar o Git como ferramenta de controlo de versões;
  2. Usar a linguagem Java como linguagem orientada a objetos;
  3. Desenhar testes unitários usando JUnit;
  4. Aplicar os princípios SOLID no desenho de aplicações e entender a sua importância;
  5. Aplicar design patterns para resolver problemas no desenho de aplicações;
  6. Identificar code smells em código orientado a objetos;
  7. Usar técnicas de refactoring para melhorar atributos não-funcionais e resolver code smells;
  8. Aplicar o padrão arquitetural MVC no desenho de aplicações event-driven;
  9. Aplicar padrões de arquitetura empresarial no desenho de aplicações.

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:

  1. Aplicar o Git como sistema de controlo de versões em projetos de software;
  2. Desenvolver aplicações orientadas a objetos usando a linguagem Java;
  3. Desenhar e implementar testes unitários usando JUnit e ferramentas de mock;
  4. Aplicar os princípios SOLID no desenho de software modular e maintível;
  5. Identificar e aplicar design patterns apropriados para resolver problemas de desenho;
  6. Identificar code smells e aplicar técnicas de refactoring para melhorar a qualidade do código;
  7. Desenhar aplicações usando o padrão arquitetural MVC;
  8. Utilizar ferramentas de suporte ao desenvolvimento e manutenção de software;
  9. Aplicar padrões de arquitetura empresarial em contextos apropriados.
