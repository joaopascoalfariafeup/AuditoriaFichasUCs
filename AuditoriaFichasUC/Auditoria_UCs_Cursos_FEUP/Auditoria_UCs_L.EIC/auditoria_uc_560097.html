Pontos fortes

  • Resultados de aprendizagem muito bem estruturados, com 13 objetivos específicos usando verbos de ação adequados (explicar, criar, mapear, identificar, etc.)
  • Sistema de avaliação equilibrado com três componentes (projeto 20%, teste SQL 30%, exame 50%) e notas mínimas apropriadas de 35%
  • Metodologias de ensino adequadas, combinando aulas teóricas com práticas em laboratório usando SQLite
  • Carga de trabalho realista totalizando exatamente 162 horas (27h × 6 ECTS) com distribuição adequada
  • Bibliografia apropriada com referências clássicas da área de bases de dados
  • Versões PT/EN coerentes e bem traduzidas
  • Condições de frequência claras incluindo assiduidade e participação no projeto

Pontos fracos

  • Programa extremamente sucinto, não permitindo compreensão clara da profundidade e sequência dos conteúdos
  • Ausência de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Objetivos gerais demasiado genéricos, focando mais no enquadramento que nos objetivos específicos da UC
  • Falta de detalhamento sobre as atividades práticas e exercícios específicos a realizar
  • Terminologia inconsistente ("appeal assessment" vs "recurso" na versão inglesa)

Sugestões de melhoria

  • Expandir significativamente o programa, detalhando cada tópico e incluindo subtópicos específicos (ex: "Diagrama de classes UML: associações, agregações, heranças, multiplicidades")
  • Adicionar mapeamento explícito entre tópicos programáticos e resultados de aprendizagem usando notação (→O1, O2, etc.)
  • Reformular os objetivos gerais para serem mais específicos e focados nos conhecimentos e competências a desenvolver
  • Detalhar as metodologias de ensino, especificando tipos de exercícios práticos e atividades de laboratório
  • Corrigir "appeal assessment" para "resit assessment" na versão inglesa
  • Considerar adicionar informação sobre ferramentas complementares além do SQLite

PROGRAMA

São abordados os seguintes tópicos programáticos, mapeados para os objetivos de aprendizagem:

  • T1. Introdução aos sistemas de gestão de bases de dados: características principais, arquitetura, vantagens e limitações (→O1)
  • T2. Modelação conceptual: diagramas de classes UML para bases de dados, entidades, atributos, associações, multiplicidades, heranças (→O2)
  • T3. Modelo relacional: conceitos fundamentais, esquemas relacionais, mapeamento de UML para relacional (→O3)
  • T4. Normalização: dependências funcionais, formas normais (1NF, 2NF, 3NF, BCNF), decomposição de esquemas (→O4, O5, O6)
  • T5. SQL - Linguagem de definição de dados: criação de tabelas, restrições de integridade, chaves primárias e estrangeiras (→O7)
  • T6. Álgebra relacional: operadores básicos (seleção, projeção, junção, união, interseção, diferença) (→O8)
  • T7. SQL - Linguagem de manipulação de dados: consultas simples e complexas, subconsultas, funções de agregação, agrupamento (→O9)
  • T8. Otimização e índices: necessidade de índices, tipos de índices, estratégias de otimização (→O10)
  • T9. Transações e concorrência: propriedades ACID, controlo de concorrência, bloqueios (→O11)
  • T10. Segurança e controlo de acesso: autenticação, autorização, privilégios em SQL, vistas, gatilhos (→O12)
  • T11. Extensões: introdução a armazéns de dados multidimensionais e sistemas NoSQL (→O13)

OBJETIVOS

Enquadramento

Esta unidade curricular aborda os fundamentos teóricos e práticos dos sistemas de bases de dados relacionais, preparando os estudantes para projetar, implementar e gerir bases de dados eficientes e seguras.

Objetivos Específicos

Os estudantes devem adquirir competências sólidas em modelação conceptual, normalização, linguagem SQL, otimização de consultas e gestão de bases de dados. A UC combina uma componente científica (50%) focada nos fundamentos teóricos com uma componente tecnológica (50%) centrada na aplicação prática usando sistemas de gestão de bases de dados.
