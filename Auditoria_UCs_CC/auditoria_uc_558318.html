Pontos fortes

  • Objetivos muito bem definidos e estruturados, com progressão lógica dos conhecimentos gerais aos específicos
  • Resultados de aprendizagem detalhados e verificáveis, usando verbos de ação adequados (identificar, compreender, conceber, planear, etc.)
  • Programa bem estruturado em blocos temáticos coerentes, com progressão lógica dos fundamentos às aplicações práticas
  • Cobertura equilibrada entre aspetos teóricos (redes, Internet) e práticos (HTML, CSS, design)
  • Bibliografia adequada e diversificada, incluindo referências clássicas e atuais
  • Software de apoio apropriado para os objetivos práticos da UC
  • Carga de trabalho realista (162h para 6 ECTS) com distribuição adequada entre componentes

Pontos fracos

  • Metodologias de ensino descritas de forma muito sucinta, sem detalhar como se alinham com os diferentes objetivos
  • Sistema de avaliação inadequado para uma UC predominantemente prática: exame final com peso de 35% é excessivo
  • Ausência de notas mínimas claramente especificadas para as componentes de avaliação
  • Condições de frequência confusas, misturando critérios de acesso (assiduidade) com critérios de aprovação (40% em cada componente)
  • Falta de clareza sobre quais componentes admitem avaliação de recurso
  • Inconsistências menores entre versões PT/EN (erros tipográficos na versão inglesa)

Sugestões de melhoria

  • Expandir a descrição das metodologias de ensino, especificando como as diferentes atividades (exposição, exercícios, acompanhamento de projetos) se alinham com os objetivos específicos
  • Reduzir drasticamente o peso do exame final (máximo 20-25%) numa UC predominantemente prática, transferindo peso para as componentes práticas
  • Especificar claramente as notas mínimas para cada componente de avaliação (ex: "Projeto Final: 45%, nota mínima 8.0 valores")
  • Separar claramente as condições de frequência (acesso ao exame) dos critérios de aprovação: "Para frequência: assiduidade ≥75% e participação nos trabalhos práticos"
  • Clarificar quais componentes admitem melhoria em recurso e quais requerem nova inscrição
  • Corrigir erros tipográficos na versão inglesa ("hof" → "how", "suport" → "support", "oerview" → "overview")
  • Considerar incluir mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem, seguindo o exemplo da ficha de referência

Proposta de Nova Redação

Métodos de ensino e atividades de aprendizagem

As aulas teórico-práticas estruturam-se em duas componentes complementares: (1) exposição e discussão dos conteúdos teóricos com recurso a meios audiovisuais, focando os fundamentos conceptuais (→objetivos 1-4, 7-8); (2) realização de exercícios práticos pelos estudantes sobre HTML, CSS, design de informação e desenvolvimento web (→objetivos 5-6, 9-11). As aulas incluem ainda sessões de acompanhamento e discussão dos projetos em desenvolvimento, promovendo a aplicação integrada dos conhecimentos teóricos e práticos adquiridos.

Fórmula de cálculo da classificação final

A avaliação compreende as seguintes componentes:

1) Avaliação distribuída (75%):
- Atividades semanais (Informática): 10%, sem nota mínima
- Análise de Sítio Web (Informática + Design): 20%, nota mínima 8.0 valores
- Projeto Final (Informática + Design): 45%, nota mínima 8.0 valores

2) Exame final: 25%, nota mínima 8.0 valores

Os trabalhos práticos são realizados em grupo (estudantes da mesma turma), mantendo-se os grupos ao longo do semestre. A classificação inclui heteroavaliação pelos pares (10%) e avaliação docente (90%).

Classificação final = 0.10×Atividades + 0.20×Análise + 0.45×Projeto + 0.25×Exame (arredondada à unidade)

Obtenção de frequência

Para acesso ao exame final, os estudantes devem:
- Cumprir a regra de assiduidade (presença em pelo menos 75% das aulas, exceto casos previstos na lei);
- Participar na realização dos trabalhos práticos (Análise de Sítio Web e Projeto Final).
