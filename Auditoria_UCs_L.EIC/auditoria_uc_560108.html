Pontos fortes

  • Resultados de aprendizagem muito bem estruturados com 10 competências específicas, detalhadas e verificáveis, adequadas ao contexto prático da UC
  • Sistema de avaliação robusto e bem pensado, com múltiplas componentes que refletem adequadamente a natureza prática da UC
  • Condições claras para obtenção de frequência e melhoria de classificação, incluindo salvaguardas contra trabalhos não participativos
  • Metodologia de ensino apropriada para UC laboratorial, combinando orientação teórica com desenvolvimento prático supervisionado
  • Carga de trabalho realista (162h para 6 ECTS) com distribuição adequada entre aulas e trabalho autónomo
  • Bibliografia adequada cobrindo bases de dados, tecnologias web e usabilidade
  • Software especificado é atual e relevante para desenvolvimento web moderno

Pontos fracos

  • Ausência de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Descrição das metodologias de ensino poderia ser mais detalhada sobre como se articulam teoria e prática
  • Fórmula de avaliação complexa com múltiplas restrições que pode ser confusa para estudantes
  • Restrição "classificação final não pode exceder mais de 5 valores da classificação do teste" é pouco clara na redação

Sugestões de melhoria

  • Adicionar mapeamento explícito dos tópicos programáticos para os objetivos de aprendizagem, seguindo o modelo: "T1. Desenvolvimento de aplicações web... (→O1, O2, O8, O9, O10)"
  • Expandir a descrição das metodologias de ensino especificando como as aulas teóricas se articulam com o trabalho prático e que tipo de orientação é fornecida
  • Clarificar a redação da restrição: "A classificação final individual não pode exceder em mais de 5 valores a classificação obtida no teste individual"
  • Considerar simplificar a apresentação da fórmula de avaliação, possivelmente com um exemplo numérico para maior clareza
  • Especificar melhor o tipo de "breves introduções às tecnologias" mencionadas nas aulas teóricas

Proposta de Nova Redação

Programa

São abordados os seguintes tópicos programáticos, mapeados para os objetivos de aprendizagem:
T1. Desenvolvimento de uma aplicação web suportada por uma base de dados: levantamento de requisitos, conceção, modelação, implementação e documentação (→O1, O2, O8, O9, O10);
T2. Projeto de bases de dados relacional com recurso a linguagens de modelação de dados (UML), à linguagem SQL, à utilização de índices de desempenho e de pesquisa, de transações, e de extensões procedimentais à linguagem SQL (→O3, O4, O5, O6, O7);
T3. Identificação e manutenção de regras de negócio: na interface, na lógica de negócio e na base de dados (triggers) (→O7, O8, O9);
T4. Projeto de aplicação web com recurso a linguagens de programação e frameworks web, centradas no servidor; aprofundamento de conceitos relacionados com a arquitetura de aplicações web (→O8, O9);
T5. Desenho de interação com o utilizador: arquitetura de informação, usabilidade e acessibilidade web (→O2, O10).

Métodos de ensino e atividades de aprendizagem

As aulas teóricas (1h/semana) serão usadas para apresentação dos guiões de trabalho, discussão de exemplos representativos, análise das avaliações efetuadas e introdução às tecnologias e conceitos fundamentais (frameworks web, padrões arquiteturais, boas práticas de desenvolvimento).

As aulas práticas laboratoriais (3h/semana) serão dedicadas ao desenvolvimento supervisionado do projeto em grupo, com o docente atuando como consultor técnico. Em cada aula prática, os componentes produzidos pelo grupo são avaliados, discutidos e orientados para as fases seguintes do desenvolvimento.
