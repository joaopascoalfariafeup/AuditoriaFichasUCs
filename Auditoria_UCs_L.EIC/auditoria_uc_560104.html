Pontos fortes

  • Objetivos de aprendizagem muito bem formulados, específicos e mensuráveis, usando verbos de ação adequados (estruturar, usar, criar, perceber, analisar)
  • Cobertura abrangente e equilibrada das tecnologias web fundamentais (HTML, CSS, PHP, JavaScript, HTTP, segurança, expressões regulares, XML)
  • Sistema de avaliação simples e transparente com pesos equilibrados entre componente prática (projeto) e teórica (exame)
  • Nota mínima de 40% no exame adequada para garantir conhecimentos fundamentais
  • Regras claras para obtenção de frequência, melhoria de classificação e casos especiais
  • Carga horária realista e bem distribuída (168h total para 6 ECTS)
  • Bibliografia adequada com referências clássicas e atualizadas
  • Coerência entre versões portuguesa e inglesa

Pontos fracos

  • Programa demasiado conciso, apresentado apenas como lista de tópicos sem estrutura ou detalhe
  • Ausência de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Metodologias de ensino descritas de forma muito genérica, sem especificar estratégias pedagógicas concretas
  • Falta de informação sobre como será avaliado o projeto (critérios, componentes, trabalho individual vs. grupo)
  • Bibliografia com algumas referências desatualizadas (2011) para tecnologias web em rápida evolução

Sugestões de melhoria

  • Expandir o programa com subtópicos detalhados para cada área (ex: "HTML 5 e CSS 3: elementos semânticos, formulários, flexbox, grid, responsive design")
  • Adicionar mapeamento explícito entre tópicos programáticos e objetivos, seguindo o modelo da ficha de referência
  • Detalhar as metodologias de ensino especificando tipos de exercícios práticos, metodologias ativas utilizadas, e como se articula a componente teórica com a prática
  • Especificar critérios de avaliação do projeto: se é individual ou em grupo, componentes avaliadas, critérios de qualidade, prazos de entrega
  • Atualizar bibliografia incluindo recursos mais recentes sobre HTML5, CSS3, JavaScript moderno e frameworks atuais
  • Considerar adicionar bibliografia complementar com recursos online e tutoriais práticos
  • Especificar melhor o software utilizado (versões, ferramentas de desenvolvimento, editores recomendados)

Proposta de Nova Redação

Programa

São abordados os seguintes tópicos programáticos, mapeados para os objetivos de aprendizagem:
T1. HTML 5 e CSS 3: estrutura semântica de documentos, formulários, elementos multimédia; estilos, layout (flexbox, grid), design responsivo (→O1, O2).
T2. A linguagem PHP: sintaxe, estruturas de controlo, funções, programação orientada a objetos; acesso a bases de dados, sessões e cookies (→O3).
T3. Javascript do lado do cliente: sintaxe, manipulação do DOM, eventos, AJAX; frameworks e bibliotecas modernas (→O4).
T4. O protocolo HTTP: métodos de pedido, códigos de resposta, headers, cookies; APIs REST (→O5).
T5. Segurança Web: vulnerabilidades comuns (XSS, SQL injection, CSRF), técnicas de prevenção, autenticação e autorização (→O6).
T6. Expressões Regulares: sintaxe, metacaracteres, grupos; aplicação na validação e processamento de dados (→O7).
T7. XML e XPath: estrutura de documentos XML, namespaces; consulta e manipulação com XPath (→O8).

Métodos de ensino e atividades de aprendizagem

As aulas teóricas combinam exposição de conceitos com demonstrações práticas ao vivo, utilizando exemplos reais de desenvolvimento web. As aulas práticas incluem exercícios guiados de programação, resolução de problemas em pequenos grupos, e desenvolvimento incremental de um projeto web completo que integra todas as tecnologias abordadas. O projeto permite aos estudantes aplicar os conhecimentos adquiridos num contexto realista, desenvolvendo uma aplicação web funcional com interface responsiva, lógica de negócio em PHP, e integração com base de dados.

Fórmula de cálculo da classificação final

A avaliação compreende as seguintes componentes:

1) Projeto (P) (Peso 50%): Desenvolvimento individual de uma aplicação web completa integrando HTML5, CSS3, PHP, JavaScript e base de dados. Avaliação baseada em funcionalidade, qualidade do código, design da interface e relatório técnico. (→O1-O8)

2) Exame final (EF) (Peso 50%): Exame individual escrito cobrindo todos os tópicos programáticos, incluindo questões teóricas e exercícios práticos de programação. Nota mínima: 40%. (→O1-O8)

A classificação final (CF) é calculada como: CF = round(0.50 * P + 0.50 * EF; 0)
