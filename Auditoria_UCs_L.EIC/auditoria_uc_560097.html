Pontos fortes

  • Resultados de aprendizagem muito bem definidos com 13 objetivos específicos usando verbos de ação adequados (explicar, criar, mapear, identificar, etc.)
  • Sistema de avaliação equilibrado com três componentes (projeto 20%, teste SQL 30%, exame 50%) que reflete adequadamente a natureza teórico-prática da UC
  • Inclusão de notas mínimas (40%) em todas as componentes, garantindo competência mínima em cada área
  • Regras claras de frequência, melhoria de classificação e tratamento de estudantes de edições anteriores
  • Carga horária realista (162h para 6 ECTS) com distribuição adequada entre contacto (32%) e trabalho autónomo
  • Coerência entre versões portuguesa e inglesa da ficha
  • Bibliografia adequada com referências clássicas da área

Pontos fracos

  • Programa muito conciso, apresentado como parágrafo único sem estruturação clara dos tópicos
  • Ausência de mapeamento explícito entre tópicos programáticos e resultados de aprendizagem
  • Metodologias de ensino descritas de forma genérica, sem detalhar atividades específicas ou estratégias pedagógicas
  • Falta de informação sobre como será avaliada a componente individual no trabalho de grupo

Sugestões de melhoria

  • Reestruturar o programa em tópicos numerados e organizados, por exemplo: "T1. Modelação conceptual: Diagrama de classes UML; T2. Modelo relacional: Esquema relacional, normalização e mapeamento; T3. SQL DDL: Linguagem de definição de dados e restrições de integridade; etc."
  • Adicionar mapeamento explícito entre tópicos programáticos e resultados de aprendizagem, seguindo o exemplo da ficha de referência
  • Expandir a descrição das metodologias de ensino, especificando tipos de exercícios práticos, metodologias ativas utilizadas, e como se articula a componente teórica com a prática
  • Clarificar na fórmula de cálculo como será tratada a avaliação individual dentro do trabalho de grupo, mencionando explicitamente que "os elementos do grupo podem ter classificações individuais distintas"
  • Considerar incluir referência a ferramentas de modelação (para além do SQLite) que possam ser utilizadas na componente de desenho de bases de dados

Proposta de Nova Redação

Programa

São abordados os seguintes tópicos programáticos:
T1. Modelação conceptual: Diagrama de classes UML para bases de dados.
T2. Modelo relacional: Esquema relacional, normalização e mapeamento de UML e E-A para relacional.
T3. SQL DDL: Linguagem de definição de dados SQL e restrições de integridade.
T4. Álgebra relacional: Álgebra relacional como linguagem de interrogação.
T5. SQL DML: Linguagem de manipulação de dados SQL.
T6. Controlo de acessos: Controlo de acessos em SQL, gatilhos e vistas.
T7. Otimização: Índices e otimização de consultas.
T8. Transações: Gestão de transações e controlo de concorrência.
T9. Tópicos avançados: Introdução a armazéns de dados e sistemas NoSQL.

Métodos de ensino e atividades de aprendizagem

As aulas teóricas são utilizadas para exposição dos conceitos fundamentais do programa, com exemplos práticos de aplicação e discussão de casos de estudo. As aulas práticas decorrem em laboratórios de informática, utilizando o sistema de gestão de bases de dados SQLite, onde os estudantes resolvem exercícios progressivos de modelação, implementação e consulta de bases de dados.
O projeto de grupo permite a aplicação integrada dos conhecimentos adquiridos, desde a modelação conceptual até à implementação e teste de uma base de dados completa. São também realizadas sessões de acompanhamento do projeto para orientação e esclarecimento de dúvidas.
