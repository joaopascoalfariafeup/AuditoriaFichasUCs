Pontos fortes

  • Programa muito bem estruturado e abrangente, cobrindo adequadamente os tópicos essenciais de desenho e teste de software orientado a objetos
  • Metodologia de ensino apropriada com progressão lógica dos exercícios práticos para um projeto integrador
  • Sistema de avaliação equilibrado com múltiplas componentes (participação, projeto, provas individuais) adequado ao caráter prático da UC
  • Bibliografia adequada e atual, incluindo referências clássicas da área
  • Carga de trabalho realista e coerente com os 6 créditos ECTS (162 horas)
  • Regras claras para recurso e melhoria de classificação

Pontos fracos

  • Objetivo 9 incompleto na versão portuguesa ("Patterns of Enterprise Applications" sem verbo de ação)
  • Inconsistências menores entre versões PT/EN nos resultados de aprendizagem
  • Formatação inadequada com tags HTML desnecessárias nos objetivos e programa
  • Falta de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Metodologias de ensino poderiam ser mais detalhadas quanto às estratégias específicas utilizadas

Sugestões de melhoria

  • Completar o objetivo 9 na versão portuguesa: "Aplicar padrões de arquitetura empresarial no desenho de aplicações"
  • Remover toda a formatação HTML desnecessária dos campos "Objetivos" e "Programa", mantendo apenas texto simples
  • Harmonizar as versões PT/EN dos resultados de aprendizagem, garantindo correspondência direta entre os pontos
  • Adicionar mapeamento explícito entre tópicos programáticos e objetivos (ex: "T1. Git → O1; T3. Java → O2")
  • Detalhar mais as metodologias de ensino, especificando estratégias como "aprendizagem baseada em problemas", "peer programming", etc.
  • Considerar especificar melhor o tipo de "provas individuais em computador" (práticas, teórico-práticas, etc.)

Proposta de Nova Redação

Objetivos (versão portuguesa)

Pretende-se, nesta unidade curricular, que os alunos desenvolvam competências no desenho de aplicações usando o paradigma orientado a objetos. Pretende-se que os estudantes que obtenham aprovação à unidade curricular, consigam:

1. Usar o Git como ferramenta de controlo de versões;
2. Usar a linguagem Java como linguagem orientada a objetos;
3. Desenhar testes unitários usando JUnit;
4. Aplicar os princípios SOLID no desenho de aplicações e entender a sua importância;
5. Aplicar design patterns para resolver problemas no desenho de aplicações;
6. Identificar code smells em código orientado a objetos;
7. Usar técnicas de refactoring para melhorar atributos não-funcionais e resolver code smells;
8. Aplicar o padrão arquitetural MVC no desenho de aplicações event-driven;
9. Aplicar padrões de arquitetura empresarial no desenho de aplicações.
