Pontos fortes

  • Programa bem estruturado cobrindo as quatro áreas fundamentais (Estatística, Processamento de Sinal, Bases de Dados, Programação) essenciais para ciência de dados
  • Bibliografia adequada e atual, incluindo recursos open source
  • Carga horária realista (324h para 12 ECTS) com distribuição equilibrada entre componentes
  • Abordagem integrada através de mini-projetos que combinam as diferentes áreas
  • Uso do Python como linguagem unificadora para todas as componentes

Pontos fracos

  • Objetivos e resultados de aprendizagem vagos, sem verbos de ação específicos e verificáveis
  • Metodologias de ensino descritas de forma muito superficial
  • Sistema de avaliação extremamente confuso e contraditório entre versões PT/EN
  • Fórmulas de cálculo inconsistentes e com erros matemáticos evidentes
  • Contradição fundamental entre "avaliação distribuída sem exame final" e existência de exames por componente
  • Falta de clareza sobre como se obtém frequência e sobre condições de melhoria
  • Inconsistências entre versões portuguesa e inglesa

Sugestões de melhoria

  • Reescrever objetivos e resultados de aprendizagem usando verbos de ação específicos (ex: "calcular", "implementar", "analisar") e especificando competências verificáveis para cada área
  • Detalhar as metodologias de ensino, especificando a organização das sessões teóricas vs. práticas, o papel dos mini-projetos e como se articulam as quatro componentes
  • Reformular completamente o sistema de avaliação, decidindo claramente se há ou não exames finais por componente
  • Corrigir as fórmulas matemáticas (ex: na versão EN há divisão por 2 extra) e garantir consistência entre versões
  • Clarificar se a nota mínima de 40% se aplica a cada componente individual ou ao conjunto exame+projeto
  • Especificar claramente as condições de obtenção de frequência além das "normas gerais"
  • Harmonizar completamente as versões portuguesa e inglesa
  • Considerar incluir mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem

Proposta de Nova Redação

Objetivos

Esta unidade curricular visa proporcionar aos estudantes os conhecimentos fundamentais em quatro áreas essenciais para o Mestrado em Engenharia e Ciência de Dados: Estatística, Processamento de Sinal, Bases de Dados e Programação. Pretende-se que os estudantes desenvolvam competências integradas nestas áreas, utilizando o Python como ferramenta unificadora, preparando-os para as restantes unidades curriculares do curso.

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:
Estatística:
E1. Aplicar técnicas de estatística descritiva para caracterizar conjuntos de dados
E2. Calcular e interpretar probabilidades e distribuições de probabilidade
E3. Construir intervalos de confiança e realizar testes de hipóteses
Processamento de Sinal:
PS1. Analisar sinais discretos no domínio do tempo e da frequência
PS2. Aplicar a transformada de Fourier e transformada Z
PS3. Projetar filtros digitais básicos
Bases de Dados:
BD1. Modelar requisitos de sistemas de informação usando diagramas UML
BD2. Transformar modelos conceptuais em modelos relacionais
BD3. Implementar e interrogar bases de dados usando SQL
Programação:
P1. Desenvolver programas em Python usando estruturas de controlo e tipos de dados
P2. Utilizar bibliotecas Python (pandas, numpy, matplotlib) para análise de dados
P3. Integrar as competências das outras áreas em projetos de análise de dados

Métodos de ensino e atividades de aprendizagem

A unidade curricular organiza-se em sessões teórico-práticas que combinam exposição de conceitos fundamentais com resolução de exercícios práticos. O ensino é estruturado em torno de três mini-projetos integradores (um por área: Estatística, Bases de Dados, Processamento de Sinal), todos implementados em Python.

As sessões de trabalho incluem: (1) apresentação e discussão de conceitos teóricos; (2) demonstração prática através de exercícios resolvidos; (3) acompanhamento do desenvolvimento dos mini-projetos; (4) sessões de esclarecimento de dúvidas. Os mini-projetos permitem aplicar os conhecimentos teóricos a problemas reais de análise de dados, promovendo a integração das quatro áreas fundamentais.

Fórmula de cálculo da classificação final

A avaliação consiste na média aritmética das classificações obtidas nas três componentes (Estatística, Bases de Dados, Processamento de Sinal). Cada componente é avaliada através de um exame individual (50%) e um mini-projeto (50%). A aprovação requer classificação mínima de 8.0 valores (40%) em cada componente.

Fórmulas de cálculo:
ClassificaçãoFinal = (ClassEstat + ClassBD + ClassProcSinal) / 3

ClassEstat = 0.5 × ExameEstat + 0.5 × min(ProjEstat, ExameEstat + 5)
ClassBD = 0.5 × ExameBD + 0.5 × min(ProjBD, ExameBD + 5)
ClassProcSinal = 0.5 × ExameProcSinal + 0.5 × min(ProjProcSinal, ExameProcSinal + 5)

Nota: A função min() limita a classificação do projeto a no máximo 5 valores acima da classificação do exame, promovendo o equilíbrio entre as duas componentes.
