Pontos fortes

  • Objetivos gerais bem definidos e relevantes para a área de ciência de dados
  • Foco em competências práticas e aplicadas, preparando estudantes para desafios profissionais
  • Bibliografia atualizada e de qualidade, incluindo recursos online acessíveis
  • Carga horária equilibrada entre diferentes tipos de atividades (39h contacto, 123h autónomas)
  • Avaliação distribuída sem exame final, adequada ao caráter prático da UC
  • Coerência entre versões portuguesa e inglesa

Pontos fracos

  • Resultados de aprendizagem formulados de forma vaga, sem verbos de ação específicos e mensuráveis
  • Programa extremamente genérico, sem detalhe sobre conteúdos específicos de cada tópico
  • Ausência de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Metodologias de ensino descritas de forma muito superficial
  • Falta de detalhes sobre as componentes de avaliação (conteúdo dos mini-testes, especificações do projeto)
  • Ausência de informação sobre avaliação especial para estudantes com estatutos especiais

Sugestões de melhoria

  • Reformular os resultados de aprendizagem usando verbos de ação específicos (ex: "analisar", "implementar", "comparar", "desenvolver") e critérios mensuráveis
  • Expandir o programa com subtópicos específicos para cada área (ex: para "Processamento de linguagem natural": tokenização, análise sintática, modelos de linguagem, etc.)
  • Adicionar mapeamento explícito entre cada tópico programático e os respetivos objetivos de aprendizagem
  • Detalhar as metodologias de ensino, especificando como cada tipo de atividade contribui para objetivos específicos
  • Especificar o conteúdo e formato dos mini-testes, bem como os critérios e entregáveis do projeto
  • Incluir secção sobre avaliação especial (TE, DA) e clarificar condições de melhoria de classificação do projeto
  • Considerar adicionar bibliografia complementar para aprofundamento de tópicos específicos

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:
O1. Identificar e caracterizar diferentes tipos de dados complexos (texto, grafos, séries temporais, dados espácio-temporais) e explicar o seu impacto nas metodologias de análise.
O2. Comparar e avaliar abordagens clássicas e estado da arte para análise de cada tipo de dados complexos abordado.
O3. Implementar e configurar ferramentas e tecnologias adequadas para análise de diferentes tipos de dados complexos.
O4. Desenvolver e adaptar metodologias de análise para novos tipos e fontes de dados complexos, justificando as escolhas técnicas.

Programa

São abordados os seguintes tópicos programáticos, mapeados para os objetivos de aprendizagem:
T1. Sistemas de recomendação avançados: filtragem colaborativa, baseada em conteúdo e híbrida; avaliação de sistemas de recomendação; desafios em dados esparsos (→O1, O2, O3)
T2. Processamento de linguagem natural: pré-processamento de texto, modelos de linguagem, análise de sentimentos, extração de informação (→O1, O2, O3)
T3. Análise de redes sociais: métricas de centralidade, deteção de comunidades, análise de influência, propagação de informação (→O1, O2, O3)
T4. Análise de séries temporais: decomposição, modelos ARIMA, previsão, deteção de anomalias (→O1, O2, O3)
T5. Análise de dados espácio-temporais: autocorrelação espacial, modelos geoestatísticos, análise de padrões espácio-temporais (→O1, O2, O3)
T6. Metodologias para novos tipos de dados: estratégias de adaptação de técnicas existentes, validação experimental (→O4)

Métodos de ensino e atividades de aprendizagem

As aulas combinam exposição teórica dos conceitos fundamentais (→O1, O2) com sessões práticas de implementação usando Python e R (→O3). Os estudantes realizam exercícios teórico-práticos em cada tópico para consolidar a aprendizagem (→O1, O2, O3). A discussão de artigos científicos recentes promove o pensamento crítico e a capacidade de adaptação a novos desafios (→O4). O projeto permite a aplicação integrada dos conhecimentos a um problema real de análise de dados complexos (→O3, O4).

Fórmula de cálculo da classificação final

A avaliação compreende as seguintes componentes:
1) Mini-testes (MT) (Peso: 50%): 3-4 testes escritos individuais realizados durante o semestre, abordando os conceitos teóricos e aplicações práticas de cada tipo de dados complexos (→O1, O2, O3). Nota mínima: 8,0 valores.
2) Projeto (P) (Peso: 50%): Desenvolvimento de uma solução completa para análise de um tipo de dados complexos, incluindo implementação, experimentação e relatório técnico, realizado individualmente ou em grupos de máximo 3 estudantes (→O3, O4). Nota mínima: 8,0 valores.

Classificação final: CF = 0,50 × MT + 0,50 × P
