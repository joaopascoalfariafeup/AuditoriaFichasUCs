(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Tema relevante e atual (análise de dados em Python) com aplicação prática a dados reais (Pordata e INE)
  • Formato e-learning adequado para uma UC de 1,5 ECTS, permitindo flexibilidade na aprendizagem
  • Estrutura modular sequencial que facilita a progressão gradual dos estudantes
  • Bibliografia adequada e acessível através da biblioteca da universidade

Pontos fracos

  • Objetivos e resultados de aprendizagem formulados de forma vaga, sem verbos de ação específicos nem níveis cognitivos definidos
  • Programa extremamente genérico, sem detalhamento dos cinco módulos mencionados
  • Sistema de avaliação contraditório: menciona avaliação distribuída mas a fórmula final é "100% exame"
  • Critério de progressão entre módulos excessivamente exigente (85%), potencialmente excludente
  • Ausência de mapeamento claro entre objetivos, programa e avaliação
  • Inconsistências entre as versões portuguesa e inglesa

Sugestões de melhoria

  • Reformular os resultados de aprendizagem com verbos de ação específicos, por exemplo: "Aplicar bibliotecas Python (pandas, numpy) para manipulação de datasets", "Criar visualizações de dados usando matplotlib", "Analisar criticamente resultados obtidos a partir de dados do INE/Pordata"
  • Detalhar o programa especificando os cinco módulos mencionados, por exemplo: "Módulo 1: Fundamentos de Python (variáveis, estruturas de controlo, funções); Módulo 2: Biblioteca pandas (importação, limpeza e manipulação de dados); Módulo 3: Visualização com matplotlib; Módulo 4: Interface com Excel; Módulo 5: Projeto prático com dados INE/Pordata"
  • Clarificar o sistema de avaliação, definindo se existe realmente avaliação distribuída ou se é apenas exame final. Se houver distribuída, especificar pesos (ex: "Quizzes de progressão: 30%; Exame final: 70%")
  • Reduzir o critério de progressão entre módulos para um valor mais razoável (ex: 60% ou 70%) ou eliminar este requisito, mantendo apenas como ferramenta de autoavaliação
  • Incluir mapeamento explícito entre cada módulo do programa e os resultados de aprendizagem correspondentes
  • Harmonizar as versões portuguesa e inglesa, corrigindo inconsistências na tradução
  • Especificar melhor as "sessões de apoio e feedback", indicando frequência e modalidade
