(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Resultados de aprendizagem bem redigidos com verbos de ação específicos e mensuráveis (identificar, demonstrar, estruturar, desenvolver, definir)
  • Sistema de avaliação equilibrado com componentes diversificadas (exame 45%, trabalho prático 45%, participação 10%)
  • Notas mínimas claramente especificadas para exame (40%) e trabalho prático (50%)
  • Carga de trabalho realista e bem distribuída (162h para 6 ECTS)
  • Bibliografia adequada e acessível através de links para o catálogo da biblioteca
  • Coerência entre versões portuguesa e inglesa
  • Flexibilidade para estudantes com estatutos especiais

Pontos fracos

  • Programa extremamente genérico e superficial, sem detalhe suficiente para compreender os conteúdos específicos
  • Ausência de mapeamento explícito entre tópicos programáticos e resultados de aprendizagem
  • Mistura inadequada de informação sobre avaliação na secção "Métodos de ensino e atividades de aprendizagem"
  • Inconsistência na terminologia das aulas (referência a "aulas práticas e práticas" na secção de frequência)
  • Condições de frequência ligeiramente confusas (nota mínima de 40% no trabalho vs. 50% para aprovação)

Sugestões de melhoria

  • Expandir significativamente o programa, detalhando os conteúdos específicos de cada tópico (ex: "Anotação de Documentos" deve especificar HTML5, semântica, acessibilidade, etc.)
  • Adicionar mapeamento explícito entre tópicos programáticos e resultados de aprendizagem (usando notação →O1, →O2, etc.)
  • Remover a subsecção "Avaliação" da secção "Métodos de ensino e atividades de aprendizagem", mantendo apenas informação sobre metodologias pedagógicas
  • Corrigir a redundância "aulas práticas e práticas" para "aulas teóricas e práticas" na secção de obtenção de frequência
  • Clarificar se a nota mínima de 40% no trabalho prático para frequência é diferente da nota mínima de 50% para aprovação, ou unificar os critérios
  • Considerar adicionar mais bibliografia complementar, especialmente sobre tecnologias web mais recentes

Proposta de Nova Redação

Programa

São abordados os seguintes tópicos programáticos, mapeados para os resultados de aprendizagem:
T1. Anotação de Documentos: Fundamentos de HTML5, estrutura semântica, elementos e atributos, acessibilidade web, boas práticas de marcação (→O1, O2).
T2. Folhas de Estilo: CSS3, seletores, propriedades de layout (Flexbox, Grid), design responsivo, pré-processadores CSS (→O1, O2).
T3. Documentos Semi-Estruturados: Introdução ao XML, sintaxe e estrutura, namespaces, validação de documentos (→O3).
T4. Definição de Tipos e Estruturas de Dados: Document Type Definition (DTD), XML Schema (XSD), modelação de dados estruturados para domínios específicos (→O4).
T5. Transformação e Manipulação de Dados Semi-Estruturados: XSLT (XSL Transformations), XPath, transformação de XML para HTML/outros formatos, processamento de dados XML (→O5).

Métodos de ensino e atividades de aprendizagem

Aulas Teóricas
Apresentação de conceitos teóricos e exemplos ilustrativos.
Discussão e demonstração de casos práticos relacionados com os temas abordados.
Exemplificação online e interativa ao longo das aulas.

Aulas Práticas
Resolução de exercícios relacionados com os temas abordados nas aulas teóricas.
Desenvolvimento progressivo do trabalho prático com acompanhamento docente.
Sessões de esclarecimento de dúvidas e revisão de conceitos.
