Pontos fortes

  • Programa bem estruturado cobrindo adequadamente sistemas de informação e análise de dados
  • Bibliografia adequada e atual para a área
  • Carga de trabalho (121.5h) coerente com os créditos ECTS (4.5 × 27h = 121.5h)
  • Combinação equilibrada entre componentes teóricas e práticas
  • Uso de ferramentas software relevantes (Access, RapidMiner)

Pontos fracos

  • Resultados de aprendizagem formulados como "preparar os alunos para" em vez de competências verificáveis
  • Ausência de verbos de ação específicos nos resultados de aprendizagem
  • Metodologias de ensino descritas de forma muito genérica
  • Sistema de avaliação com regras confusas e mecanismo de limitação da classificação do trabalho pouco transparente
  • Inconsistências entre versões PT/EN (especialmente nos resultados de aprendizagem)
  • Secção "Obtenção de frequência" vaga e sem critérios específicos

Sugestões de melhoria

  • Reformular os resultados de aprendizagem usando verbos de ação específicos (ex: "analisar", "projetar", "implementar", "avaliar") em vez de "preparar os alunos para"
  • Adicionar mapeamento explícito entre tópicos programáticos e resultados de aprendizagem
  • Detalhar as metodologias de ensino, especificando estratégias pedagógicas concretas para cada tipo de conteúdo
  • Clarificar o sistema de avaliação, especialmente o mecanismo "P2 = P1 + 5" e definir critérios específicos para obtenção de frequência
  • Harmonizar as versões PT/EN, corrigindo inconsistências nos resultados de aprendizagem
  • Especificar notas mínimas para todas as componentes de avaliação ou justificar a sua ausência

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:

O1. Analisar necessidades organizacionais e identificar fatores críticos de sucesso para sistemas de informação.
O2. Aplicar técnicas de modelação conceptual de informação, nomeadamente modelação de classes de objetos com UML.
O3. Projetar bases de dados relacionais utilizando técnicas de normalização funcional.
O4. Implementar protótipos de sistemas de informação utilizando sistemas de gestão de bases de dados relacionais.
O5. Aplicar metodologias de projeto de engenharia informática no desenvolvimento de sistemas de informação.
O6. Aplicar técnicas de análise de dados e data mining para extração de conhecimento, incluindo a metodologia CRISP-DM.
O7. Construir e avaliar modelos de classificação utilizando métricas apropriadas.

Métodos de ensino e atividades de aprendizagem

A UC decorrerá com aulas teórico-práticas estruturadas da seguinte forma:
- Exposição teórica dos conceitos fundamentais seguida de discussão e análise de casos práticos
- Resolução de exercícios de aplicação em contexto de sala de aula
- Desenvolvimento de trabalhos práticos em grupo utilizando ferramentas software (MS Access/MySQL para bases de dados e R/RapidMiner para análise de dados)
- Sessões de acompanhamento e orientação dos projetos práticos
- Apresentação e discussão dos trabalhos desenvolvidos pelos estudantes

Fórmula de cálculo da classificação final

A classificação final será calculada com base nas seguintes componentes:

P1 - Exame final: 50% (Nota mínima: 6,5 valores)
P2 - Trabalho de grupo: 50%
  P2.1 - Projeto em Access: 30%
  P2.2 - Projeto de Análise de Dados: 20%

Classificação final = 0,5 × P1 + 0,5 × P2
onde P2 = 0,6 × P2.1 + 0,4 × P2.2

Nota: Para garantir coerência entre as componentes de avaliação, caso a classificação do trabalho (P2) exceda a classificação do exame (P1) em mais de 5 valores, será considerada para cálculo da classificação final P2 = P1 + 5.

Obtenção de frequência

Para obtenção de frequência, os estudantes devem:
- Cumprir a regra de assiduidade (máximo de 25% de faltas às aulas)
- Realizar e entregar os trabalhos práticos (P2.1 e P2.2)
- Responder ao inquérito de caracterização do perfil no prazo estipulado (sob pena de penalização até 25% na componente P2)
