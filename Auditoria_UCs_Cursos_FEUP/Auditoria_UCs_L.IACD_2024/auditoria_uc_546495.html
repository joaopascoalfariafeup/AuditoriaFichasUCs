Pontos Fortes

  • Programa abrangente e bem estruturado: O programa cobre de forma sistemática todos os aspetos fundamentais das estruturas de dados, desde conceitos básicos de Java até algoritmos avançados de ordenação e pesquisa.
  • Progressão lógica dos conteúdos: A sequência dos tópicos é pedagogicamente adequada, começando pelos fundamentos da linguagem e evoluindo para estruturas de dados mais complexas.
  • Equilíbrio na avaliação: O sistema combina adequadamente avaliação teórica (exame 70%) com componentes práticas (25%) e participação (5%).
  • Componente prática bem definida: As aulas práticas têm objetivos claros de apoio aos estudantes na resolução de exercícios e aplicação dos conceitos.
  • Condições de aprovação claras: São estabelecidas notas mínimas tanto no exame (≥7) como na classificação final (≥9.5).
  • Coerência entre versões: As versões em português e inglês são consistentes e bem traduzidas.

Pontos Fracos

  • Objetivos de aprendizagem mal formulados: Os objetivos não seguem a taxonomia de Bloom, usando expressões vagas como "saber usar" em vez de verbos de ação específicos e verificáveis.
  • Ausência de mapeamento explícito: Não existe mapeamento claro entre objetivos de aprendizagem, tópicos programáticos e componentes de avaliação.
  • Metodologias de ensino pouco detalhadas: Falta especificação de atividades concretas de aprendizagem e como estas contribuem para os objetivos.
  • Formatação inadequada: O uso de tags HTML nos resultados de aprendizagem prejudica a legibilidade.
  • Falta de diversidade na avaliação: Não há trabalhos de projeto ou outras formas de avaliação que promovam competências de nível cognitivo superior.

Sugestões de Melhorias

  • Reformular os objetivos de aprendizagem: Usar verbos de ação específicos da taxonomia de Bloom (ex: "implementar", "analisar", "comparar", "aplicar") e torná-los mais verificáveis.
  • Criar mapeamento explícito: Estabelecer correspondência clara entre cada objetivo de aprendizagem, tópicos programáticos e componentes de avaliação.
  • Detalhar metodologias de ensino: Especificar atividades concretas como laboratórios práticos, projetos de implementação, estudos de caso, etc.
  • Diversificar a avaliação: Considerar incluir um projeto prático de implementação de estruturas de dados para avaliar competências de síntese e aplicação.
  • Corrigir formatação: Remover tags HTML desnecessárias e melhorar a apresentação dos resultados de aprendizagem.
  • Especificar recursos de aprendizagem: Incluir bibliografia e recursos digitais que apoiem o estudo autónomo dos estudantes.
