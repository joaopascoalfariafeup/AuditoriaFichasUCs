Síntese da Avaliação

Pontos Fortes

  • Tentativa de alinhamento com o framework CDIO para definição de competências
  • Referência à utilização de métodos de aprendizagem ativa
  • Coerência entre as versões portuguesa e inglesa
  • Flexibilidade no sistema de melhoria de classificação

Pontos Fracos

  • Objetivos de aprendizagem inadequados: Não são apresentados objetivos específicos, mensuráveis e verificáveis. A descrição baseia-se em competências genéricas CDIO sem tradução em resultados de aprendizagem concretos com verbos de ação apropriados
  • Programa pouco detalhado: Os tópicos programáticos são apresentados de forma muito sintética, sem descrição adequada dos conteúdos ou contextualização
  • Metodologias de ensino vagas: A descrição das metodologias é extremamente genérica, não especificando atividades concretas nem como se relacionam com os objetivos
  • Sistema de avaliação confuso: A estrutura de avaliação é pouco clara, com componentes mal definidas (exercícios + 2 testes = 100%?) e sem mapeamento para objetivos específicos
  • Ausência de mapeamento: Não existe alinhamento explícito entre objetivos, conteúdos, metodologias e avaliação
  • Problemas técnicos: Presença de código HTML desnecessário na secção de objetivos

Sugestões de Melhorias

  • Reformular objetivos de aprendizagem: Definir 6-8 objetivos específicos usando verbos de ação da taxonomia de Bloom (ex: "Formular modelos de programação linear para problemas de otimização", "Aplicar o método simplex para resolver problemas de programação linear")
  • Detalhar o programa: Expandir cada tópico com descrição dos conteúdos específicos e sua relevância para a engenharia de materiais
  • Especificar metodologias: Descrever atividades concretas de aprendizagem ativa, exercícios práticos e como se relacionam com cada objetivo
  • Clarificar sistema de avaliação: Definir claramente as componentes (percentagens não somam 100%), estabelecer notas mínimas quando aplicável, e mapear cada componente para objetivos específicos
  • Criar mapeamento explícito: Estabelecer correspondência clara entre objetivos, tópicos, metodologias e avaliação
  • Limpar formatação: Remover código HTML desnecessário e melhorar a apresentação geral da ficha
