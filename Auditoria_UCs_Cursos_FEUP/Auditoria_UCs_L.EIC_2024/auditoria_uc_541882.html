Pontos Fortes

  • Programa abrangente: Os tópicos programáticos cobrem de forma adequada as principais áreas da engenharia de software, desde processos até verificação e validação
  • Metodologia equilibrada: Combina adequadamente teoria e prática através de um projeto de desenvolvimento em grupo
  • Sistema de avaliação diversificado: Inclui múltiplas componentes (exame, trabalho escrito, trabalho prático, participação) com pesos equilibrados
  • Componente supervisionada: Inclui exame final individual (35%) que garante avaliação supervisionada
  • Projeto prático relevante: O projeto de desenvolvimento com relatório e código no GitHub é adequado à natureza da disciplina
  • Flexibilidade para estudantes especiais: Prevê adaptações para estudantes dispensados de assiduidade

Pontos Fracos

  • Objetivos de aprendizagem inadequados: Formulados de forma vaga e genérica, sem verbos de ação específicos da taxonomia de Bloom
  • Ausência de mapeamentos: Não existe mapeamento explícito entre objetivos, tópicos programáticos e métodos de avaliação
  • Falta de detalhamento na avaliação: Não especifica claramente como cada componente avalia diferentes objetivos ou níveis cognitivos
  • Condições de frequência imprecisas: Critérios vagos como "participação e contribuição efetiva" sem definição clara
  • Inconsistências entre versões: Pequenas discrepâncias entre as versões portuguesa e inglesa (ex: fórmula de cálculo)

Sugestões de Melhorias

  • Reformular objetivos de aprendizagem: Usar verbos de ação específicos (ex: "aplicar", "analisar", "avaliar") e torná-los mais específicos e verificáveis
  • Criar mapeamentos explícitos: Estabelecer correspondência clara entre objetivos, tópicos programáticos e componentes de avaliação
  • Detalhar critérios de avaliação: Especificar como cada componente avalia diferentes objetivos e níveis cognitivos
  • Clarificar condições de frequência: Definir critérios objetivos e mensuráveis para "participação efetiva"
  • Harmonizar versões linguísticas: Corrigir inconsistências entre as versões portuguesa e inglesa
  • Especificar notas mínimas: Clarificar se existem notas mínimas para as componentes de trabalho prático
  • Melhorar descrição das metodologias: Explicitar como as diferentes atividades contribuem para objetivos específicos
