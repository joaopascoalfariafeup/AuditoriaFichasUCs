Estruturas de Dados e Algoritmos

Apreciação da Ficha da Unidade Curricular

Aspeto a Avaliar Apreciação Qualitativa Apreciação Quantitativa (0–3) (a)
Objetivos/resultados de aprendizagem Objetivos genéricos e resultados de aprendizagem pouco específicos, sem verbos de ação claros nem mapeamento explícito para níveis cognitivos 1
Tópicos programáticos Programa bem estruturado e detalhado, cobrindo adequadamente programação orientada por objetos e estruturas de dados 2
Metodologias de ensino e aprendizagem Metodologias adequadas combinando teoria e prática, mas descrição pouco detalhada sobre o mapeamento para os objetivos 2
Metodologias de avaliação Sistema de avaliação simples mas adequado, com componentes equilibradas e condições claras de aprovação 2
Apreciação global Ficha com estrutura sólida mas necessita de melhorias na especificidade dos objetivos de aprendizagem 1.75

(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Programa bem estruturado em duas partes lógicas (programação orientada por objetos e estruturas de dados/algoritmos)
  • Tópicos programáticos detalhados e adequados ao nível da unidade curricular
  • Sistema de avaliação equilibrado (40% distribuída + 60% exame) com condições claras de aprovação
  • Metodologias de ensino apropriadas combinando aulas teórico-práticas e práticas
  • Carga horária coerente com os 6 créditos ECTS (162 horas totais)
  • Flexibilidade para estudantes manterem nota de frequência do ano anterior
  • Versões portuguesa e inglesa coerentes entre si

Pontos fracos

  • Objetivos de aprendizagem vagos e genéricos, sem verbos de ação específicos da taxonomia de Bloom
  • Ausência de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Falta de detalhamento sobre como as metodologias de ensino contribuem para cada objetivo
  • Componentes de avaliação não mapeadas explicitamente para objetivos específicos
  • Descrição muito genérica do trabalho prático (apenas "trabalho prático")

Sugestões de melhoria

  • Reformular os resultados de aprendizagem usando verbos de ação específicos, por exemplo: "O1. Implementar classes em Python aplicando conceitos de encapsulamento e herança", "O2. Analisar a complexidade temporal de algoritmos usando notação Big-O", "O3. Selecionar estruturas de dados apropriadas (listas, pilhas, filas, árvores) para resolver problemas específicos"
  • Adicionar mapeamento explícito no programa, indicando quais tópicos contribuem para cada objetivo (ex: "Conceitos fundamentais de classes (O1)", "Análise da complexidade de algoritmos (O2)")
  • Detalhar melhor o trabalho prático, especificando que tipo de projeto será desenvolvido e como será avaliado (ex: "Desenvolvimento de um projeto de programação em Python envolvendo implementação de estruturas de dados e algoritmos para resolver um problema de bioengenharia")
  • Incluir nas metodologias de ensino como cada tipo de aula contribui para objetivos específicos (ex: "Aulas teórico-práticas focadas nos objetivos O1 e O2", "Aulas práticas direcionadas para os objetivos O2 e O3")
  • Especificar melhor as componentes de avaliação indicando quais objetivos cada uma avalia (ex: "Trabalho prático: avalia objetivos O1, O2 e O3", "Exame: avalia todos os objetivos O1-O3")
