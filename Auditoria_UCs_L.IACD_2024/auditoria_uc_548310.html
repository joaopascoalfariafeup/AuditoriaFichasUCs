Tecnologias Web

Apreciação da Ficha da Unidade Curricular

Aspeto a Avaliar Apreciação Qualitativa Apreciação Quantitativa (0–3) (a)
Objetivos/resultados de aprendizagem Objetivos vagos e genéricos, sem verbos de ação específicos nem mapeamento claro para níveis cognitivos 1
Tópicos programáticos Lista básica de tecnologias sem estruturação pedagógica ou sequenciamento lógico 1
Metodologias de ensino e aprendizagem Descrição muito genérica sem detalhamento das atividades ou estratégias específicas 1
Metodologias de avaliação Sistema simples mas adequado, com componentes equilibradas e critérios mínimos definidos 2
Apreciação global Ficha com estrutura básica mas necessita de maior detalhamento pedagógico 1.25

(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Sistema de avaliação equilibrado com componente prática significativa (40%) e teórica (60%), adequado à natureza da UC
  • Definição clara de critérios mínimos (25%) tanto para testes como para trabalho prático
  • Trabalho em grupo com dimensão adequada (até 3 estudantes) que permite colaboração sem comprometer a avaliação individual
  • Carga horária coerente com os 6 créditos ECTS (162 horas totais)
  • Coerência entre as versões portuguesa e inglesa da ficha

Pontos fracos

  • Objetivos de aprendizagem excessivamente vagos e genéricos, sem verbos de ação específicos nem níveis cognitivos definidos
  • Programa apresentado como lista simples de tecnologias, sem estruturação pedagógica ou sequenciamento lógico
  • Metodologias de ensino descritas de forma muito genérica, sem detalhamento das estratégias pedagógicas
  • Ausência de mapeamento entre objetivos, programa e avaliação
  • Bibliografia desatualizada (livros de 2005-2011) para uma área tecnológica em rápida evolução

Sugestões de melhoria

  • Reformular os resultados de aprendizagem com verbos de ação específicos, por exemplo: "O1. Construir páginas web estruturadas utilizando HTML5 e CSS3", "O2. Implementar interatividade client-side usando JavaScript e DOM", "O3. Desenvolver comunicação assíncrona entre cliente e servidor usando AJAX", "O4. Criar aplicações web server-side usando Node.js"
  • Reestruturar o programa em módulos sequenciais com mapeamento para objetivos: "M1. Fundamentos web: HTML5, CSS3 e protocolo HTTP (O1)", "M2. Programação client-side: JavaScript, DOM e APIs HTML5 (O2)", "M3. Comunicação assíncrona: AJAX e APIs REST (O3)", "M4. Desenvolvimento server-side: Node.js e alternativas (O4)"
  • Detalhar as metodologias de ensino especificando: "Aulas teóricas com demonstrações práticas de código, aulas laboratoriais com exercícios guiados, desenvolvimento incremental de projeto prático, revisão de código em pares"
  • Incluir mapeamento explícito entre componentes de avaliação e objetivos de aprendizagem na fórmula de cálculo
  • Atualizar a bibliografia incluindo recursos mais recentes e online, como documentação MDN Web Docs, tutoriais oficiais do Node.js, e livros atualizados sobre desenvolvimento web moderno
