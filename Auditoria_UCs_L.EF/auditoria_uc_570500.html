Objetivos/resultados de aprendizagem Objetivos claros e adequados ao nível introdutório, mas resultados de aprendizagem vagos e sem verbos de ação específicos da taxonomia de Bloom 2
Tópicos programáticos Programa bem estruturado e adequado para uma UC introdutória de programação, mas sem mapeamento explícito para os objetivos 2
Metodologias de ensino e aprendizagem Descrição muito genérica e pouco detalhada das metodologias, não permitindo compreensão clara das atividades 1
Metodologias de avaliação Sistema de avaliação adequado com componente supervisionada forte, mas descrição pouco clara dos "testes com correção automática" 2

(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Objetivos bem definidos e apropriados para uma UC introdutória de programação em Engenharia Física
  • Programa estruturado de forma lógica, cobrindo os conceitos fundamentais de programação em Python
  • Bibliografia adequada e acessível, com recursos online gratuitos
  • Sistema de avaliação com forte componente supervisionada (80% entre testes e exame final)
  • Carga horária realista e bem distribuída (162h para 6 ECTS)
  • Coerência entre versões portuguesa e inglesa

Pontos fracos

  • Resultados de aprendizagem vagos, sem verbos de ação específicos e mensuráveis
  • Ausência de mapeamento explícito entre tópicos programáticos e objetivos
  • Metodologias de ensino descritas de forma muito genérica
  • Descrição pouco clara dos "testes com correção automática" na avaliação
  • Condições de frequência pouco precisas ("pelo menos metade dos testes")

Sugestões de melhoria

  • Reformular os resultados de aprendizagem usando verbos de ação específicos (ex: "identificar", "implementar", "analisar", "criar")
  • Adicionar mapeamento explícito entre cada tópico programático e os objetivos correspondentes
  • Detalhar as metodologias de ensino, especificando tipos de exercícios, projetos práticos, e como se articulam aulas teóricas e práticas
  • Clarificar a natureza dos "testes com correção automática": frequência, duração, tipo de questões
  • Especificar com precisão as condições de frequência (ex: "realizar com sucesso pelo menos 50% dos testes de cada componente")
  • Considerar incluir uma pequena componente de projeto prático individual para consolidar aprendizagens

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:
R1. Identificar e utilizar os componentes básicos do sistema operativo GNU/Linux para desenvolvimento de programas.
R2. Implementar algoritmos elementares utilizando as estruturas de controlo fundamentais da linguagem Python.
R3. Criar programas estruturados aplicando boas práticas de programação e documentação.
R4. Aplicar estruturas de dados fundamentais (listas, dicionários, tuplos) na resolução de problemas.
R5. Utilizar a biblioteca numpy para operações com arrays e cálculos numéricos básicos.
R6. Analisar e corrigir erros de programação através de técnicas de debugging e testing.
R7. Desenvolver programas que manipulem ficheiros e utilizem módulos da biblioteca padrão Python.

Métodos de ensino e atividades de aprendizagem

A UC combina aulas teóricas (2h/semana) e práticas (2h/semana):
- Aulas teóricas: exposição dos conceitos fundamentais, análise de código exemplificativo, e discussão de boas práticas de programação.
- Aulas práticas: resolução guiada de exercícios de programação, desenvolvimento de pequenos projetos, e realização de testes práticos com correção automática.
- Trabalho autónomo: estudo da bibliografia, resolução de exercícios adicionais, e desenvolvimento de programas para consolidação das competências adquiridas.

Obtenção de frequência

Para acesso ao exame final, os estudantes têm de:
- Cumprir a regra de assiduidade (não faltar a mais de 25% das aulas práticas);
- Realizar com sucesso pelo menos 50% dos testes práticos com correção automática, tanto na componente teórica como na componente prática.
