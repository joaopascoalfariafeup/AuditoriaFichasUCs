  • Corrigir erros ortográficos na versão inglesa ("pittfalls" → "pitfalls", "secuirty" → "security")

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:
RA1. Analisar e comparar arquiteturas end-to-end para aplicações móveis, incluindo componentes de backend.
RA2. Desenvolver aplicações móveis nativas para Android utilizando as ferramentas e frameworks adequados.
RA3. Implementar mecanismos de segurança em aplicações móveis, incluindo gestão de permissões e KeyChains.
RA4. Aplicar os princípios do RGPD no desenvolvimento de aplicações móveis, garantindo privacidade e consentimento informado.
RA5. Otimizar aplicações móveis considerando trade-offs entre performance, consumo energético e segurança.
RA6. Utilizar técnicas de programação de baixo nível (C/C++, JNI, NDK) para melhorar performance e portabilidade.
RA7. Implementar comunicações device-to-device utilizando diferentes tecnologias (WiFi-Direct, Bluetooth, etc.).
RA8. Aplicar técnicas de profiling e debugging para identificar e resolver problemas em aplicações móveis.

Métodos de ensino e atividades de aprendizagem

A UC combina aulas teóricas expositivas com sessões práticas laboratoriais intensivas:
- Aulas teóricas (1/3 do tempo): Apresentação e discussão dos conceitos fundamentais, arquiteturas, e boas práticas de desenvolvimento móvel.
- Aulas práticas laboratoriais (2/3 do tempo): Desenvolvimento hands-on de aplicações Android, com exercícios progressivos que culminam num projeto integrador.
- Projeto prático: Desenvolvimento de uma aplicação móvel completa que integra os conceitos abordados, incluindo backend, segurança, e otimização energética.
- Sessões de code review e debugging colaborativo para desenvolver competências de análise crítica de código.
- Seminários sobre casos de estudo reais da indústria, incluindo análise de aplicações populares e suas arquiteturas.

Programa

1. Fundamentos de Desenvolvimento Móvel (→RA1, RA2)
1.1. Arquiteturas end-to-end para aplicações móveis
1.2. Arquiteturas de hardware de smartphones (BigLittle, etc.)
1.3. Ciclo de vida das aplicações móveis
1.4. Frameworks de desenvolvimento móvel

2. Desenvolvimento Android (→RA2, RA6)
2.1. Estruturas de aplicações Android
2.2. Programação de baixo nível: JNI + NDK
2.3. Integração com backends: HTTP(s) e gRPC

3. Segurança e Privacidade (→RA3, RA4)
3.1. Aspectos de segurança em aplicações móveis
3.2. Sistema de permissões e KeyChains
3.3. RGPD: consentimento informado e proteção de dados
3.4. Requisitos para publicação no Google Play Store

4. Otimização e Performance (→RA5, RA8)
4.1. Gestão energética: Doze mode e otimizações
4.2. Push notifications eficientes
4.3. Profiling e debugging de aplicações
4.4. Trade-offs: performance vs. energia vs. segurança

5. Comunicações Avançadas (→RA7)
5.1. Comunicações device-to-device (D2D)
5.2. WiFi-Direct, Bluetooth, TDLS
5.3. Integração com redes WiFi e 4G LTE
