(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Programa muito completo e bem estruturado, cobrindo todos os aspetos fundamentais de bases de dados relacionais
  • Resultados de aprendizagem organizados de forma clara em competências de "Saber" e "Fazer"
  • Avaliação equilibrada com componentes diversificadas (projeto, teste prático, exame) adequadas à natureza da UC
  • Inclusão de avaliação prática supervisionada (teste SQL em laboratório) que garante integridade do processo
  • Bibliografia adequada e atual para a área
  • Carga de trabalho coerente com os 6 ECTS (162 horas)

Pontos fracos

  • Objetivos demasiado genéricos e pouco específicos para uma UC de 3º ano
  • Alguns resultados de aprendizagem usam verbos pouco mensuráveis ("conhecer", "caracterizar")
  • Metodologias de ensino descritas de forma muito superficial, sem explicar como se articulam teoria e prática
  • Ausência de mapeamento explícito entre tópicos programáticos e objetivos de aprendizagem
  • Falta de clarificação sobre possibilidade de melhoria das componentes de avaliação distribuída
  • Inconsistências menores entre versões PT/EN (ex: "muluvalue" em vez de "multivalued")

Sugestões de melhoria

  • Reformular os objetivos para serem mais específicos, substituindo "Dotar os estudantes da teoria e prática necessária" por objetivos mais concretos como "Capacitar os estudantes para projetar, implementar e otimizar bases de dados relacionais"
  • Nos resultados de aprendizagem, substituir verbos como "conhecer" e "caracterizar" por verbos mais específicos como "identificar", "aplicar", "analisar", "avaliar"
  • Expandir a descrição das metodologias de ensino, explicando como as aulas práticas complementam a teoria e que tipo de exercícios/projetos são realizados
  • Adicionar mapeamento explícito entre tópicos programáticos e objetivos (ex: "T1. Introdução aos sistemas de bases de dados (→O1, O2)")
  • Clarificar na secção de avaliação que as componentes de avaliação distribuída podem ser melhoradas por nova frequência da UC no ano seguinte
  • Corrigir erros tipográficos na versão inglesa ("muluvalue" → "multivalued")

Proposta de Nova Redação

Objetivos

Capacitar os estudantes para projetar, implementar e otimizar bases de dados relacionais, fornecendo os conhecimentos teóricos e competências práticas necessários para desenvolver soluções eficazes de gestão de dados em contextos profissionais.

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:

Conhecimentos:

  • O1. Identificar as características e funcionalidades de um sistema de gestão de base de dados;
  • O2. Aplicar o modelo de classes UML e o modelo relacional de dados;
  • O3. Utilizar a álgebra relacional como linguagem formal de manipulação de bases de dados relacionais;
  • O4. Dominar os fundamentos da linguagem SQL para definição, manipulação, interrogação e controlo de acesso de dados;
  • O5. Aplicar regras de normalização de bases de dados com base em dependências funcionais e multivalor.

Competências:

  • O6. Projetar uma base de dados de média dimensão em UML e converter para base de dados relacional;
  • O7. Implementar e manipular uma base de dados utilizando SQL;
  • O8. Desenvolver aplicações com transações e controlo de concorrência.

Métodos de ensino e atividades de aprendizagem

A UC combina aulas teóricas e práticas numa abordagem integrada:

Aulas Teóricas: Exposição e discussão dos conceitos fundamentais de bases de dados, incluindo modelos de dados, álgebra relacional, SQL e normalização. Utilização de exemplos práticos para ilustrar os conceitos teóricos.

Aulas Práticas: Resolução de exercícios de modelação, implementação de bases de dados em SQL, e desenvolvimento de um projeto prático em grupo. As aulas práticas incluem trabalho laboratorial com o MySQL Server para consolidar os conhecimentos teóricos através da implementação prática.

O projeto de grupo permite aos estudantes aplicar todos os conceitos aprendidos num contexto realista, desde a modelação conceptual até à implementação e otimização da base de dados.
