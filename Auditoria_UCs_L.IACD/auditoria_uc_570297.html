Pontos fortes

  • Carga horária (162h) coerente com os 6 ECTS atribuídos
  • Distribuição equilibrada entre horas de contacto (30%) e trabalho autónomo
  • Bibliografia adequada e atual para a área de tecnologias web
  • Avaliação distribuída apropriada para UC prática, incluindo trabalho em grupo
  • Coerência geral entre versões PT e EN (exceto nas metodologias de avaliação)

Pontos fracos

  • Objetivos demasiado vagos e genéricos, sem especificidade técnica
  • Resultados de aprendizagem não usam verbos de ação verificáveis da taxonomia de Bloom
  • Programa apresentado como lista simples sem estruturação pedagógica
  • Metodologias de ensino descritas de forma muito genérica
  • Inconsistências graves entre versões PT/EN na avaliação (8+12 vs 25%)
  • Confusão entre critérios de frequência e aprovação
  • Falta de mapeamento entre tópicos programáticos e objetivos

Sugestões de melhoria

  • Reformular objetivos para serem específicos: "Desenvolver aplicações web usando HTML5, CSS3 e JavaScript" em vez de "familiarizar com conceitos"
  • Reescrever resultados de aprendizagem com verbos específicos: "Implementar", "Configurar", "Desenvolver", "Analisar", seguindo a taxonomia de Bloom
  • Estruturar o programa em módulos sequenciais com indicação de horas e mapeamento para objetivos
  • Detalhar metodologias de ensino especificando tipos de exercícios práticos e como se articulam com a teoria
  • Corrigir inconsistências na avaliação: uniformizar pesos entre versões PT/EN
  • Clarificar que a nota mínima de 25% é para aprovação, não para frequência
  • Separar claramente critérios de frequência (participação) dos critérios de aprovação (notas mínimas)
  • Especificar melhor as componentes de avaliação: quantos testes, quando, que tópicos cobrem

Proposta de Nova Redação

Objetivos

Esta unidade curricular tem como objetivo capacitar os estudantes para o desenvolvimento de aplicações web modernas, abrangendo tecnologias client-side e server-side. Os estudantes irão dominar as linguagens fundamentais da web (HTML5, CSS3, JavaScript), compreender a arquitetura e protocolos web, e implementar soluções completas usando Node.js.

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:
O1. Implementar interfaces web responsivas usando HTML5 e CSS3 avançado.
O2. Desenvolver funcionalidades interativas client-side com JavaScript e manipulação do DOM.
O3. Configurar comunicação assíncrona entre cliente e servidor usando AJAX e APIs REST.
O4. Analisar e aplicar o protocolo HTTP em aplicações web.
O5. Construir servidores web dinâmicos usando Node.js e frameworks associados.
O6. Integrar APIs do HTML5 em aplicações web modernas.
O7. Avaliar e implementar diferentes arquiteturas para aplicações web escaláveis.

Programa

T1. Fundamentos de marcação e apresentação web (20h → O1):
- HTML5: elementos semânticos, formulários avançados, validação
- CSS3: layouts responsivos, flexbox, grid, animações

T2. Programação client-side (20h → O2, O6):
- JavaScript: ES6+, manipulação do DOM, eventos
- APIs do HTML5: localStorage, geolocalização, canvas

T3. Comunicação web (8h → O3, O4):
- Protocolo HTTP: métodos, códigos de estado, headers
- AJAX: XMLHttpRequest, Fetch API, JSON

T4. Desenvolvimento server-side (20h → O5, O7):
- Node.js: módulos, npm, programação assíncrona
- Frameworks web: Express.js, routing, middleware
- Modelos alternativos: CGI, comparação de arquiteturas

Fórmula de cálculo da classificação final

A avaliação é distribuída sem exame final, compreendendo:

1) Testes teóricos (T1, T2) - Peso: 60%
- Dois testes individuais durante o semestre
- Cobertura: T1 (tópicos T1-T2), T2 (tópicos T3-T4)

2) Trabalho prático (TP) - Peso: 40%
- Desenvolvimento de aplicação web completa em grupos de 2-3 estudantes
- Entrega em 3 fases: frontend, backend, integração

Classificação final: CF = 0.6 × (T1+T2)/2 + 0.4 × TP

Nota mínima para aprovação: 8.0 valores em cada componente (testes e trabalho).

Obtenção de frequência

Para obtenção de frequência, os estudantes devem:
- Cumprir a regra de assiduidade (máximo 25% de faltas às aulas práticas obrigatórias)
- Participar em pelo menos um teste teórico
- Realizar pelo menos uma entrega do trabalho prático
