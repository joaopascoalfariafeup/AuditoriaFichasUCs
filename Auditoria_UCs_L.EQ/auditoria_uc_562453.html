Pontos fortes

  • Abordagem e-learning adequada ao contexto de uma UC de 1,5 ECTS
  • Foco em competências práticas relevantes (Python para análise de dados)
  • Bibliografia adequada e acessível através da biblioteca da universidade
  • Carga horária coerente com os créditos ECTS (40,5h para 1,5 ECTS)
  • Utilização de dados reais (Pordata e INE) para contextualizar a aprendizagem

Pontos fracos

  • Sistema de avaliação contraditório: declara "avaliação distribuída com exame final" mas a fórmula é 100% exame
  • Critério de 85% para progressão entre módulos é excessivamente elevado e pode criar barreiras desnecessárias
  • Objetivos e resultados de aprendizagem formulados de forma vaga, sem verbos de ação específicos
  • Programa muito superficial, sem detalhe suficiente sobre os conteúdos específicos
  • Ausência de mapeamento entre objetivos, programa e avaliação
  • Inconsistências entre versões PT e EN (formatação HTML desnecessária na versão EN)

Sugestões de melhoria

  • Clarificar o sistema de avaliação: se a avaliação distribuída (quizzes) não conta para a nota final, alterar o tipo de avaliação para "Exame final" ou incluir peso dos quizzes na fórmula
  • Reduzir o critério de progressão de 85% para 60-70%, mais adequado para permitir aprendizagem progressiva
  • Reformular os resultados de aprendizagem com verbos de ação específicos (ex: "aplicar", "implementar", "analisar") e níveis cognitivos claros
  • Detalhar o programa especificando conteúdos concretos para cada módulo (ex: estruturas de dados, manipulação de DataFrames, visualização de dados)
  • Incluir mapeamento explícito entre objetivos, tópicos programáticos e componentes de avaliação
  • Uniformizar a formatação entre versões PT e EN, removendo tags HTML desnecessárias
  • Especificar melhor as "sessões de apoio e feedback" nas metodologias de ensino

Proposta de Nova Redação

Resultados de aprendizagem e competências

No final da unidade curricular, os estudantes devem ser capazes de:
R1. Aplicar os conceitos fundamentais de programação em Python para resolução de problemas de análise de dados.
R2. Utilizar bibliotecas especializadas (pandas, numpy, matplotlib) para manipulação, análise e visualização de dados.
R3. Implementar interfaces entre Python e folhas de cálculo (MS-Excel) para importação e exportação de dados.
R4. Analisar séries de dados reais provenientes de fontes oficiais (Pordata, INE) aplicando técnicas apropriadas.
R5. Resolver problemas práticos de gestão e análise de grandes volumes de dados utilizando Python.

Programa

O programa está estruturado em cinco módulos sequenciais:
Módulo 1: Fundamentos de programação em Python (→R1)
- Sintaxe básica, estruturas de dados, controlo de fluxo
- Funções e módulos
Módulo 2: Bibliotecas para análise de dados (→R2)
- Numpy: arrays e operações matemáticas
- Pandas: manipulação de DataFrames e Series
- Matplotlib: visualização de dados
Módulo 3: Interface com folhas de cálculo (→R3)
- Leitura e escrita de ficheiros Excel
- Automatização de tarefas em Excel
Módulo 4: Análise de dados reais (→R4)
- Acesso a dados da Pordata e INE
- Limpeza e preparação de dados
Módulo 5: Projeto prático (→R5)
- Resolução de problema completo de análise de dados

Fórmula de cálculo da classificação final

A avaliação compreende as seguintes componentes:
1) Avaliação distribuída (AD) - Peso: 30%
- Quizzes online de progressão entre módulos (nota mínima: 60% em cada quiz para acesso ao módulo seguinte)
2) Exame final (EF) - Peso: 70%
- Exame prático em sala de computadores (1h30m, nota mínima: 45%)
Classificação final: CF = 0,30 × AD + 0,70 × EF (arredondada à unidade)

Obtenção de frequência

Para acesso ao exame final, os estudantes devem:
- Completar todos os módulos da UC (obter classificação ≥ 60% em todos os quizzes de progressão)
- Cumprir a regra de assiduidade nas sessões de apoio obrigatórias (quando aplicável)
