Metodologias de ensino e aprendizagem Metodologia adequada combinando teoria e prática. Descrição clara da abordagem pedagógica, embora possa beneficiar de maior detalhe sobre estratégias específicas de aprendizagem ativa. 2
Metodologias de avaliação Sistema de avaliação distribuída apropriado para UC prática, mas com algumas lacunas: ausência de notas mínimas claras, componente DP pouco detalhada, e falta de clarificação sobre avaliação de recurso para todas as componentes. 1
Apreciação global Ficha globalmente adequada para UC introdutória de programação, com estrutura sólida mas necessitando de melhorias na clarificação do sistema de avaliação e maior precisão em alguns aspetos. 1.75

(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Resultados de aprendizagem bem estruturados com verbos de ação apropriados (projetar, implementar, testar, entender, definir)
  • Programa abrangente e bem sequenciado, desde conceitos básicos até tópicos avançados como testes unitários
  • Bibliografia adequada e atual, incluindo recursos clássicos e modernos sobre Python
  • Metodologia de ensino apropriada combinando aulas teóricas e práticas, adequada para uma UC introdutória
  • Carga de trabalho realista (162h para 6 ECTS) com distribuição equilibrada entre contacto (35%) e estudo autónomo
  • Coerência entre versões portuguesa e inglesa
  • Sistema de avaliação distribuída apropriado para uma UC prática sem exame final

Pontos fracos

  • Objetivos demasiado genéricos, sem especificidade suficiente para orientar o ensino
  • Ausência de mapeamento explícito entre tópicos programáticos e resultados de aprendizagem
  • Componente "desempenho nas aulas práticas" (DP) insuficientemente detalhada
  • Ausência de notas mínimas para as componentes de avaliação
  • Falta de clarificação sobre possibilidade de avaliação de recurso para componentes DP e μT
  • Terminologia inconsistente (micro-testes vs minitestes)

Sugestões de melhoria

  • Reformular os objetivos para serem mais específicos, por exemplo: "Desenvolver competências de programação em Python para resolver problemas típicos da Engenharia Aeroespacial, incluindo cálculos numéricos, processamento de dados e automação de tarefas"
  • Adicionar mapeamento explícito entre tópicos programáticos e resultados de aprendizagem, usando notação como (→O1, O2)
  • Detalhar a componente DP especificando critérios: "Desempenho nas aulas práticas, avaliado com base na participação ativa, qualidade dos exercícios realizados e progressão na resolução de problemas"
  • Incluir notas mínimas para as componentes principais, por exemplo: "Componente MT (nota mínima: 45%)"
  • Clarificar as possibilidades de avaliação de recurso: especificar se as componentes DP e μT podem ser recuperadas e como
  • Uniformizar a terminologia, optando por "minitestes" em todas as referências
  • Adicionar informação sobre como é calculada a componente DP (critérios, frequência de avaliação, etc.)

Proposta de Nova Redação

Objetivos

Desenvolver competências fundamentais de programação em Python que servirão de base para a resolução de problemas computacionais em Engenharia Aeroespacial. Especificamente, pretende-se que os estudantes dominem os conceitos e técnicas de programação necessários para automatizar cálculos, processar dados e implementar algoritmos básicos, preparando-os para unidades curriculares mais avançadas que requerem competências de programação.

Programa

São abordados os seguintes tópicos programáticos, mapeados para os resultados de aprendizagem:

T1. Introdução ao Pensamento Computacional, algoritmos e programação com Python (→O1, O2).

T2. Conceitos fundamentais de programação: tipos de dados simples; variáveis, expressões e declarações; fluxo de programa, condicionais, iteração (→O1, O2).

T3. Funções, passagem de parâmetros, recursão; ficheiros (→O1, O3).

T4. Tipos de dados compostos: strings, tuplas, conjuntos, listas, dicionários (→O2).

T5. A biblioteca "standard" de Python (→O4).

T6. Listas e coleções; iteradores e geradores (→O5).

T7. Estratégias de resolução de problemas (→O1, O2, O3).

T8. Ferramentas de programação, depuração e exceções (→O1).

T9. Testes unitários (→O6).

Fórmula de cálculo da classificação final

A classificação final é dada pela seguinte fórmula:

Classificação final = 0.1 × DP + 0.2 × μT + 0.7 × MT

onde:

Componente DP = Desempenho nas aulas práticas, avaliado com base na participação ativa, qualidade dos exercícios realizados e progressão na resolução de problemas (Peso: 10%)

Componente μT = 0.5 × μT1 + 0.5 × μT2, sendo μT1 e μT2 os minitestes teóricos realizados em sala de aula (Peso: 20%)

Componente MT = 0.5 × MT1 + 0.5 × MT2, sendo MT1 e MT2 os minitestes práticos realizados em salas de exame (Peso: 70%; Nota mínima: 45%)

Melhoria de classificação

Os estudantes que frequentaram esta unidade curricular no presente ano letivo podem fazer melhoria das componentes MT1 e MT2 na época de recurso, na forma de um único exame que substitui por inteiro a componente MT.

As componentes DP e μT não admitem melhoria de classificação, devendo ser repetidas numa nova frequência da unidade curricular.
